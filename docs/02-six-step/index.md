# 方法篇：六步推理链

> **一条线性递进的思考链——每一步的输出是下一步的输入。跳步 = 返工。**

---

## 核心循环

```
业务产出物 → 能力拆解 → 功能映射 → 设计推理 → 分步操作 → 验收
   ↑_________________________________________↓
                  （验收不过→回溯到对应步骤）
```

**底部信条**：学会一个场景，就会搭任意一个。六步框架不变，变的是每步的填充内容。

---

## 六步概览

| 步骤 | 核心问题 | 产出 |
|---|---|---|
| **Step 1** 业务产出物 | 我最终要交付什么？ | 应用定义模板（用户/场景/交付物/成功标准） |
| **Step 2** 能力拆解 | 要产出它，需要哪几种能力？ | 能力清单 + 优先级（P0/P1/P2） |
| **Step 3** 功能映射 | 每种能力对应平台哪个功能？选哪种模式？ | 技能映射表 + 模式选择结论 |
| **Step 4** 设计推理 | 为什么选它？边界与兜底在哪？ | 四维论证（选它/不选/边界/兜底）+ 搭建模式推荐结论 |
| **Step 5** 分步操作 | 怎么搭？ | 操作步骤清单（按模式分支） |
| **Step 6** 验收 | 做完怎么验对错？常见坑怎么躲？ | 验收用例 + 坑位清单 |
