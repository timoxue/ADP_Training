# Skills 与提示词框架

---

## Skills 是什么

Skill 是封装好的**可复用能力包**。

你不需要写代码——从 Skills 广场找到对应技能，点击安装，Agent 就会这个能力了。

```
没有 Skill            有了 Skill
────────────          ──────────────────
Agent 只能说话   →    Agent 还能查行情、读文档、发消息
```

### Skills 的触发方式

Agent 会根据用户的意图自动判断要不要调用某个 Skill：

- 用户问"宁德时代今天收盘多少" → 自动调用 `a-stock-data`
- 用户发来一段营销话术 → 自动调用 `securities-compliance-check`
- 用户说"帮我发到企业微信群" → 自动调用 `wecom-unified`

每个 Skill 的 **description 字段**就是它的"触发说明"，Agent 通过读 description 判断何时调用。

---

## Claw 模式与标准模式的区别

| 项目 | Claw 模式 | 标准 / 工作流 / Multi-Agent |
|------|-----------|---------------------------|
| 挂载方式 | Skills 广场即装即用 | 连接器 + 工作流节点 |
| 最大数量 | 80 个 Skills | 无硬性限制 |
| 合规硬拦截 | ❌ 无内容安全等设置 | ✅ 有完整安全设置 |
| 适合场景 | 灵活探索、数据分析 | 严肃业务、合规要求高的场景 |

## 提示词框架

ADP 内置 AI 提示词优化工具，也支持手动按框架编写。下面是两套最常用框架。

---

### 框架 A：标准模式（对话助手）

适用于知识问答、产品咨询、投研助手等对话场景。

```
## 角色
你是一个[职责描述]，专注于[核心能力]，服务对象为[用户群体]。

## 风格特点
1. [语气/专业度要求]
2. [输出特点]
3. [引用/数据规范]

## 输出要求
回复必须包含以下模块，标题用【】标注：
1. 【核心观点】：...
2. 【关键数据】：...（标注出处）
3. 【风险提示】：...

## 能力限制
1. 仅基于知识库内容回答，不引用外部信息
2. 若知识库中无相关内容，直接告知用户"暂无相关资料"
3. 不涉及[禁止事项]
4. 结尾固定附：本内容由 AI 基于知识库生成，仅供内部参考，不构成投资建议。
```

---

### 框架 B：Agent / Multi-Agent 模式

适用于有工具调用、多步骤执行的 Agent。

```
# 任务目标
作为[角色]，你需要[核心职责]。

# 任务流程
1. [步骤一，含工具调用说明]
2. [步骤二]
3. [步骤三]

# 限制
- [约束一]
- [约束二]
```

Multi-Agent 的每个子 Agent 还需要额外声明：

- **转交描述**：什么情况下调度 Agent 会把任务转给你
- **输入来源**：数据从哪个上游 Agent 接收
- **输出目标**：结果交给谁

## 平台变量

提示词中可直接引用平台变量，无需写代码：

| 变量 | 含义 | 使用场景 |
|------|------|---------|
| `{{SYS.UserQuery}}` | 本轮用户输入原文 | 工作流节点中引用用户原始问题 |
| `{{SYS.CurrentTime}}` | 当前时间 | 让 Agent 知道"今天"是什么时候 |
| `{{SYS.Memory}}` | 跨会话长期记忆内容 | 个性化场景，记住用户偏好 |
| `{{API.XXX}}` | 调用方传入的自定义参数 | 如 `{{API.UserName}}` 亲切称呼用户 |
| `{{APP.XXX}}` | 工作流间共享变量 | 工作流 A 赋值，工作流 B 读取 |

## adp-prompt-builder 引导流程

ADP 平台提供 `adp-prompt-builder` 技能，通过对话式引导帮你一步步写好提示词。**每次只问一个问题，不跳步。**

### Step 1 — 确认应用模式

> 你的 ADP 应用是哪种模式？
> - 标准模式（对话助手，无工具调用）
> - Agent 模式（有工具/插件调用）
> - Multi-Agent 模式（多个子 Agent 协作）

### Step 2 — 收集现有内容

> 有提示词草稿就贴出来；没有就描述：这个助手帮谁用、做什么。

### Step 3 — 诊断缺失元素

| 要素 | 标准模式（框架 A） | Agent 模式（框架 B） |
|------|-------------------|---------------------|
| 身份定义 | `## 角色` | `# 任务目标` |
| 执行逻辑 | `## 风格特点` | `# 任务流程` |
| 输出规范 | `## 输出要求` | 在 `# 限制` 里说明 |
| 边界约束 | `## 能力限制` | `# 限制` |

### Step 4 — 逐一补全缺失项（每次只问一项）

- 角色/目标缺失 → "这个助手主要帮用户解决什么问题？"
- 风格缺失 → "希望助手以什么语气回复？有引用规范要求吗？"
- 输出格式缺失 → "回复必须包含哪些内容模块？"
- 限制缺失 → "知识库外的问题如何处理？有哪些内容绝对不能回答？"

### Step 4.5 — 个性化变量确认（可选）

需要在对话中用到用户姓名、当前时间、或跨会话记忆吗？

- 需要 → 使用 `{{API.UserName}}`、`{{SYS.CurrentTime}}`、`{{SYS.Memory}}`
- 不需要 → 跳过

### Step 4.6 — 行业合规确认（证券场景必做）

应用是否服务于受监管行业（证券/银行/医疗/法律）？

**→ 证券/投研场景** 逐一确认：

1. **合规声明格式**：是否有公司规定的固定免责声明？没有则使用默认：
   > `本内容由 AI 基于知识库生成，仅供内部参考，不构成投资建议。`

2. **保守回答**：知识库外的问题应直接告知"暂无相关资料"，并在平台「回复设置」中开启**保守回答**双重保障。

3. **客户分级**（可选）：不同角色需要不同深度的内容吗？需要则通过 `{{API.ClientType}}` 传入角色类型。

### Step 5 — 输出完整提示词

按对应框架输出，代码块包裹，可直接粘贴进「角色指令」文本框。

### Step 6 — 平台功能提示

| 功能 | 位置 | 用途 |
|------|------|------|
| AI 一键优化 | 角色指令 → 一键优化 | 自动将草稿结构化 |
| 保存为模板 | 角色指令 → 模板 | 团队复用，按任务/行业打标签 |
| 版本记录 | 角色指令 → 版本 | 保存历史版本，支持对比 |
| 保守回答 | 应用设置 → 回复设置 | 知识库外问题走固定话术 |

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| 模型不遵循提示词格式 | 用分层标题拆开每项，避免连续大段文字 |
| 知识库外的问题被乱回答 | 在"能力限制"写兜底话术，或开启「保守回答」 |
| 工具调用时机不准确 | 在任务流程中明确"何时调用什么工具" |
| Multi-Agent 转交混乱 | 每个子 Agent 的转交描述必须明确且互不重叠 |
| 输出格式不稳定 | 用【标题】格式锁定结构 |
| 多轮对话丢失上下文 | 在「模型设置」中调整上下文轮数，不需要写进提示词 |
| 想个性化称呼用户 | 提示词写 `{{API.UserName}}`，调用方通过 API 参数传入 |

---

!!! tip "证券场景必加的三条限制"
    1. **不承诺收益**：禁止"保本""稳赚""一定涨"等词出现在回复中
    2. **不代客操作**：不接入任何交易类工具
    3. **不构成投资建议**：超出知识库范围时，引导联系持牌投顾

    这三条写进「能力限制」是软约束；要做到硬拦截，还需在应用设置中开启「内容安全」。
