# LLM · RAG · Agent

> **本节目标**：理解三个核心概念，以及它们在 ADP 平台中如何协同工作。

---

## 大语言模型（LLM）

LLM（Large Language Model）是一切 AI 应用的"大脑"。

| 特性 | 说明 |
|------|------|
| 训练数据截止 | 模型知识有时效性，无法感知最新信息 |
| 上下文窗口 | 每次对话能"记住"的内容有上限 |
| 幻觉问题 | 模型可能生成听起来合理但错误的内容 |

!!! warning "证券场景的核心挑战"
    LLM 的"幻觉"在证券场景中风险极高。**RAG 是解决方案之一**。

---

## 检索增强生成（RAG）

RAG（Retrieval-Augmented Generation）= 先检索，再生成。

```
用户提问
    ↓
从知识库检索相关片段
    ↓
将片段 + 问题一起发给 LLM
    ↓
LLM 基于真实材料生成回答
```

**ADP 平台中的对应功能**：知识库 → 知识检索节点 → 大模型知识问答节点

---

## 智能体（Agent）

Agent = LLM + 工具调用能力 + 自主规划

```
接收任务
    ↓
分析需要哪些工具/步骤
    ↓
调用工具（搜索、计算、查询数据库...）
    ↓
综合结果，生成回答
```

**与普通 LLM 对话的区别**：Agent 会主动"做事"，而不只是"说话"。

---

## 三者关系

```
LLM（思考）
  + RAG（知识）
  + Agent（行动）
= ADP 智能体应用
```
