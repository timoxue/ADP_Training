# 四种应用模式

ADP 平台支持四种应用模式，从简单到复杂依次对应不同的业务场景。

---

## 模式对比总览

| 模式 | 核心特点 | 适合场景 | 本课程对应 |
|------|---------|---------|-----------|
| **Claw 模式** | 类 Claude/ChatGPT，自动选择工具 | 通用问答、探索型任务 | — |
| **标准模式** | 固定系统提示词，单轮/多轮对话 | 客服、问答、内容生成 | 场景一·场景二 |
| **单工作流模式** | 可视化流程编排，节点串联 | 有明确步骤的自动化任务 | 场景三·场景四 |
| **Multi-Agent 模式** | 多个 Agent 协作，主 Agent 调度 | 复杂多步骤、需交叉校验 | 进阶篇 |

---

## Claw 模式

> 最像"原生 AI 助手"的模式。

- 模型自主决定何时调用哪个工具
- 无需手动配置工作流
- 适合探索期、需求不明确的场景

---

## 标准模式

> 最简单、最常用的起点。

- 核心配置：**角色指令（提示词）**
- 可接入知识库实现 RAG
- 支持多轮对话上下文

```
用户输入 → [系统提示词约束] → 模型回复
                ↑
           （可选）知识库检索
```

---

## 单工作流模式

> 把业务流程"画"出来。

- 可视化节点编排：开始 → 处理 → 判断 → 回复
- 支持条件判断、循环、代码执行
- 每个节点可独立调试

!!! example "合规风控示例"
    话术检测 → 标签提取节点 → 条件判断（高/中/低风险）→ 回复节点输出清单

---

## Multi-Agent 模式

> 一个"主 Agent" 协调多个"子 Agent"。

- 主 Agent：接活、分工、汇总
- 子 Agent：各司其职（检索/计算/审核/撰写）
- 关键配置：**转交关系**（最常见错误：忘配转交关系）

```
主 Agent（调度）
├── 资料检索 Agent
├── 数据分析 Agent  
├── 风控合规 Agent
└── 结论撰写 Agent
```

!!! warning "常见错误"
    忘记配置子 Agent 的**转交描述**，导致主 Agent 不知道何时转给谁。

---

!!! tip "如何选择模式"
    - 纯问答/客服 → **标准模式**
    - 有明确流程步骤 → **单工作流模式**
    - 需要多角色交叉验证 → **Multi-Agent 模式**
    - 不确定需求 → **Claw 模式** 先探索
