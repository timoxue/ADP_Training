# ADP 平台功能总览

> 腾讯云智能体开发平台（Agent Development Platform）

---

## 平台是怎么长出来的

ADP 不是一夜之间出现的，而是从实际业务需求一层层迭代来的：

| 阶段 | 核心能力 | 解决的问题 |
|------|---------|-----------|
| 第一阶段 | 知识库 + RAG 问答 | 大模型幻觉、接不进企业数据 |
| 第二阶段 | 知识引擎 + 低代码应用 | 业务人员自助搭问答应用 |
| 第三阶段 | Agent + 工作流 + Multi-Agent + MCP | 工具调用、流程自动化、多角色协同 |

---

## 一句话记住分层

```
ADP 平台  =  办公大楼     （承载、管控、运维）
Agent    =  数字员工     （会思考、会调度）
Skill    =  工具人       （被调用的能力）
MCP 网关 =  前台         （唯一对外收口）
```

## 六大核心模块

### 1. 应用管理

创建和运营 AI 应用的核心入口。

- 支持四种模式：Claw / 标准 / 单工作流 / Multi-Agent
- 每个应用独立配置：提示词、知识库、技能、工具、合规设置
- 发布管理：测试环境 → 正式发布 → 版本回滚

### 2. 知识库

企业内部文档的向量化存储与检索系统。

- 支持格式：PDF、Word、Excel、TXT、Markdown 等
- 文档状态：导入中 → 解析中 → 审核中 → 学习中 → 导入完成
- 检索配置：混合检索、Top-K（建议 5–10）、相似度阈值（建议 0.5–0.7）
- 开启引用溯源后，回答会标注"来自第 X 页"

### 3. 工作流

可视化的多步骤流程编排工具。

常用节点类型：

| 节点 | 用途 |
|------|------|
| 开始节点 | 接收用户输入或触发条件 |
| 知识检索节点 | 从知识库拉取相关文档片段 |
| 大模型节点 | 调用 LLM 进行分析、摘要、分类 |
| 条件判断节点 | 根据结果走不同分支（高/中/低风险） |
| 代码节点 | 运行 Python/JS 处理数据 |
| HTTP 请求节点 | 调用外部 API |
| 群机器人节点 | 推送到企业微信/钉钉 Webhook |
| 结束节点 | 返回最终结果 |

### 4. 技能（Skills）与连接器

扩展 Agent 能力边界的组件。详见 [Skills 说明](skills-harness.md)。

- **Skills**：封装好的能力包，即装即用（Claw 模式，最多 80 个）
- **连接器**：标准模式/工作流模式下对接外部系统的接口

### 5. 安全合规

平台级的内容治理机制，独立于提示词，无法被绕过。

| 功能 | 说明 | 在哪配置 |
|------|------|---------|
| 内容安全 | 关键词命中直接替换为处置文案 | 应用设置 → 内容安全 |
| 保守回答 | 知识库外的问题返回固定话术 | 应用设置 → 回复设置 |
| 隐私脱敏 | 身份证/银行卡/手机号自动打码 | 应用设置 → 隐私脱敏 |
| 长期记忆 | 跨会话记住用户偏好 | 应用设置 → 变量与记忆 |

!!! warning "Claw 模式无此设置"
    内容安全、保守回答、隐私脱敏仅在**标准模式 / 单工作流 / Multi-Agent** 中可用。  
    Claw 模式没有这些设置项，合规风控只能靠提示词软约束。

### 6. 平台管理

企业级运营所需的管控能力。

- **工作空间隔离**：不同事业部/项目数据独立，互不可见
- **操作日志**：全量留痕，满足监管审查要求
- **发布审批**：上线需多人决策流程
- **告警与监控**：接口异常、调用量超限即时告警

## 六大治理机制

金融机构落地 ADP 的合规底座：

| 机制 | 说明 |
|------|------|
| **数据不出域** | 私有化部署，数据留内网 |
| **信息隔离墙** | 工作空间数据独立隔离 |
| **投资者适当性** | 固化风险匹配与风险揭示逻辑 |
| **内容合规** | 安全策略拦截违规承诺（硬拦截） |
| **留痕可审计** | 操作日志、对话记录、版本回滚 |
| **发布审批** | 上线需多人决策，上线即合规 |

## 应用设置关键参数

| 参数 | 位置 | 说明 |
|------|------|------|
| 上下文轮数 | 应用设置 → 模型设置 | 控制多轮对话的记忆长度，默认较短 |
| 知识库检索参数 | 知识 → 检索配置 | Top-K、相似度、混合检索开关 |
| 欢迎语 | 应用设置 → 对话体验 | 用户打开应用看到的第一句话 |
| 开场问题 | 应用设置 → 对话体验 | 快捷问题，最多 3 条 |
| 系统变量 | 提示词中直接引用 | `{{SYS.CurrentTime}}`、`{{SYS.Memory}}` 等 |
