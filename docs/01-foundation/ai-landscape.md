# 主流模型与框架速览

> 市场上的 AI 产品眼花缭乱。这一页帮你建立基本的选型判断力。

---

## 主流大语言模型

### 国际主流

| 模型 | 厂商 | 特点 | 适合场景 |
|------|------|------|---------|
| **GPT-4o** | OpenAI | 综合能力强，生态最成熟，多模态 | 通用问答、代码生成 |
| **Claude 3.5 / 4** | Anthropic | 长上下文、指令遵循好、安全性高 | 长文档分析、结构化输出 |
| **Gemini 1.5 Pro** | Google | 超长上下文（100万 token），多模态强 | 超长文档、视频理解 |
| **Llama 3** | Meta | 开源免费，可私有部署 | 本地化部署、成本敏感场景 |

### 国内主流

| 模型 | 厂商 | 特点 | 适合场景 |
|------|------|------|---------|
| **DeepSeek-V3 / R1** | 深度求索 | 开源、成本极低、推理能力强 | 数据分析、代码、数学推理 |
| **Kimi K2** | 月之暗面 | 长上下文、中文理解好 | 长研报分析、中文写作 |
| **混元** | 腾讯 | ADP 平台默认模型，企业集成好 | ADP 平台内应用 |
| **文心 4.0** | 百度 | 中文知识丰富 | 中文内容生成 |
| **通义千问** | 阿里 | 开源版本丰富，API 生态全 | 代码、多轮对话 |

### 选型基本原则

1. **数据不出境要求** → 优先国内模型或私有化部署（Llama、DeepSeek 开源版）
2. **长文档处理** → 看上下文窗口大小（Gemini 100万，Claude 20万）
3. **成本敏感** → DeepSeek API 价格约为 GPT-4o 的 1/30
4. **合规要求高** → 优先有企业级合规认证的商用 API

## 主流 Agent 框架

### 如何看待框架的演进

可以用**注意力的五次演进**来理解市场上的框架：

| 演进 | 框架/工具 | 解决的问题 |
|------|---------|----------|
| 工具注意力 | **MCP 协议** | 给大模型装"手"——统一接入外部工具和数据源 |
| 持续注意力 | **Claude Code / Codex** | 给大模型装"纪律"——持续追踪任务状态，自主执行 |
| 开放注意力 | **OpenClaw** | 让人人都能搭——开源可私有化，技能可复用 |
| 结构化注意力 | **Hermes Agent** | 给大模型装"记忆"——四层分层记忆，越用越懂你 |
| 组织注意力 | **企业级底座（ADP 等）** | 从个人工具走向企业落地——统一管控、审计、合规 |

### MCP 协议

**Model Context Protocol** — AI 调用外部工具的标准接口。

类比：USB 接口。不同设备插上就能用，MCP 让大模型和外部工具有了统一的"对话方式"。

目前面向金融场景的 MCP 工具已超过 **186 个**，覆盖行情数据、财报获取、公告检索、估值模型等主要投研需求。

### Claude Code

命令行 Agent，运行在你的电脑或服务器上，能：

- 读文件、写代码、调命令、跑结果
- 遇到报错自己纠正，一步一步交付
- 持续追踪整个任务状态（不是一问一答）

核心是一个 while 循环：思考 → 行动 → 观察结果 → 再思考 → 再行动……直到完成。

**适合投研场景**："查数据 → 建模 → 核对 → 成文"这个流程，以前要多次来回，现在描述目标，一气呵成。

### OpenClaw（龙虾）

开源 Agent 框架，核心价值三点：

- **Skills 技能体系**：能力封装成可复用"技能包"，可沉淀、可共享
- **Gateway 常驻进程**：默认 18789 端口，7×24 在线，可定时触发
- **开源可私有化**：数据留在自己机房，满足监管要求

养一只"虾"三步：① 定人格 → ② 立规矩 → ③ 建能力

### Hermes Agent

自进化 AI 智能体框架，GitHub 11 万+ Star：

- **自我进化**：完成任务后自动提炼成功经验为可复用技能
- **四层记忆**：冻结提示 / 技能 / 会话记忆 / 长期记忆，跨会话记住你
- **轻量**：内存占用比 OpenClaw 最高低 72.7%，支持 200+ 模型

### 框架选型口诀

```
只想问答       →  知识库 + RAG
固定流程       →  工作流
复杂多角色     →  Multi-Agent
开发者自建     →  OpenClaw / Hermes
券商合规落地   →  企业级底座（ADP）
```

## 选模型 vs 选框架

两件事不要混淆：

| | 模型（Model） | 框架（Framework） |
|--|------------|----------------|
| 是什么 | 大脑，负责理解和生成 | 骨架，负责调度和执行 |
| 换了影响 | 回答质量、风格、成本 | 工作流程、工具调用方式 |
| 典型产品 | GPT-4o、Claude、DeepSeek | ADP、OpenClaw、Hermes |
| 换一个难吗 | 通常只改 API 配置 | 可能需要重新搭建应用 |

好的企业级底座（如 ADP）应该支持切换底层模型，让你不被单一供应商锁定。

## 关于幻觉的补充

幻觉是目前所有 LLM 共同的问题，没有模型能完全避免。对金融机构来说，三条防线缺一不可：

1. **RAG**：让模型基于真实材料生成，而不是凭空推断
2. **人工审核节点**：AI 的输出在进入业务流程前经过人工复核
3. **留痕可审计**：所有 AI 生成的内容有日志，出问题能追溯
