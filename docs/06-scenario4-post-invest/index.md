# 场景四：投后经营

> **持续盯持仓相关公告和舆情，自动抓取、生成简报、推到工作群**

**痛点**：投后跟踪依赖人工刷新闻，信息遗漏风险高；简报撰写重复劳动。  
**方案**：单工作流模式（Fetch → 大模型 → 群机器人 Webhook）

## 工作流结构

```
Fetch 插件（抓取公告）→ 大模型节点（生成简报）→ Webhook（推送到企业微信/钉钉/飞书）
```
