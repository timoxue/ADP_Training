# 场景一：监管文件智能解析

> 合规部 / 法务部 / 风控部 · 把一份通知拆解成可执行清单

---

## 使用场景

收到监管通知或内部合规文件后，需要：梳理哪些条款有截止日期、各部门分别负责什么、生成执行摘要发给相关负责人。以下演示用一份虚构的合规通知作为示例文件。

---

## 示例文件

将以下内容保存为 `合规通知节选.md`，放在工作目录中。

??? note "展开查看示例文件内容"

    ```markdown
    # 关于进一步加强证券公司信息安全与合规管理工作的通知

    **发文机关**：公司合规管理委员会
    **文号**：合规委〔2024〕18号
    **发文日期**：2024年9月10日
    **执行部门**：全体业务及职能部门

    ## 一、总体要求

    各部门须严格落实"合规经营、风险可控"原则，将合规管理工作纳入日常业务流程。各部门负责人为本部门合规管理第一责任人，须于每季度末向合规部提交合规自查报告。

    ## 二、客户信息保护

    **第2.1条** 任何部门及员工不得将客户身份信息、账户信息、交易记录以任何形式对外传输或存储于未经审批的第三方系统。违规者将依据《员工行为准则》第十二条处理，情节严重的移送监管机关。

    **第2.2条** 涉及客户数据的系统上线前，须由信息技术部完成安全评估并出具书面意见，合规部复核后方可投入使用。评估周期不少于10个工作日。

    **第2.3条** 各部门应于**2024年10月31日前**完成客户信息台账梳理，并报合规部备案。超期未报的，依规对部门负责人进行通报批评。

    ## 三、内幕信息管理

    **第3.1条** 投资银行部、研究所及资产管理部人员涉及未公开重大信息时，须在24小时内向合规部进行信息登记，登记表格见附件一。

    **第3.2条** 上述部门人员在信息公开前，不得买卖相关证券，其直系亲属同样受此约束。合规部将对上述人员账户实施动态监控。

    **第3.3条** 各部门应建立物理或逻辑隔离措施，防止内幕信息跨部门流转。信息技术部须于**2024年12月15日前**完成信息屏障系统升级工作。

    ## 四、员工行为管理

    **第4.1条** 员工从事与公司业务存在竞争关系的兼职活动，须事先获得人力资源部及合规部双重审批。未经审批擅自从事兼职的，一经发现立即终止劳动合同。

    **第4.2条** 员工接受客户赠礼的价值上限为**500元/次，年累计不超过2000元**，超出部分须上缴公司，并在礼品登记系统中如实填报。

    **第4.3条** 禁止使用个人社交媒体账号发布与公司业务相关的市场分析、投资建议等内容。违规发布须在合规部要求的时限内删除，并接受相应处分。

    ## 五、反洗钱管理

    **第5.1条** 客户服务部、经纪业务部须对存量客户进行风险等级重新评定，评定结果于**2025年1月31日前**录入系统。高风险客户须进行加强型尽职调查（EDD）。

    **第5.2条** 单笔交易超过人民币**50万元**或等值外币的，须在交易完成后1个工作日内完成大额交易报告。

    **第5.3条** 员工发现可疑交易须第一时间向合规部报告，不得向客户或任何第三方透露已提交可疑交易报告的事实。

    ## 六、检查与问责

    **第6.1条** 合规部将于每年第一季度开展年度合规专项检查，各部门须配合提供相关材料，不得拒绝或阻挠。

    **第6.2条** 检查发现的问题，相关部门须在30个工作日内完成整改并提交整改报告。逾期未整改的，将对部门负责人予以扣分处理，并纳入年度绩效考核。

    **第6.3条** 本通知自发文之日起执行，解释权归合规管理委员会。
    ```

## 操作步骤

### 步骤 1：启动 Claude Code

```bash
claude
```

### 步骤 2：提取截止日期任务清单

```
请读取当前目录下的 合规通知节选.md，找出所有有明确截止日期或时间要求的条款，
按截止时间从早到晚排列，用表格展示：条款编号 / 截止时间 / 要求内容 / 责任部门
```

**预期输出**：

| 条款编号 | 截止时间 | 要求内容 | 责任部门 |
|---------|---------|---------|---------|
| 第2.3条 | 2024年10月31日 | 完成客户信息台账梳理并报合规部备案 | 全体部门 |
| 第3.3条 | 2024年12月15日 | 完成信息屏障系统升级工作 | 信息技术部 |
| 第5.1条 | 2025年1月31日 | 完成存量客户风险等级重新评定并录入系统 | 客户服务部、经纪业务部 |
| 第3.1条 | 24小时内（持续） | 未公开重大信息须向合规部登记 | 投行部、研究所、资管部 |
| 第5.2条 | 1个工作日内（持续） | 单笔超50万元须提交大额交易报告 | 所有业务部门 |

---

### 步骤 3：按部门拆解责任

```
再把整份通知按责任部门重新整理一遍，每个部门有哪些具体要执行的事项，
格式：部门名称 → 事项清单（带条款号）
```

**预期输出**（节选）：

**信息技术部**
- 系统上线前完成安全评估并出具书面意见（第2.2条）
- 2024年12月15日前完成信息屏障系统升级（第3.3条）

**投资银行部 / 研究所 / 资产管理部**
- 涉及未公开重大信息须24小时内向合规部登记（第3.1条）
- 信息公开前禁止买卖相关证券（第3.2条）

---

### 步骤 4：生成执行摘要邮件

```
帮我写一份给各部门负责人的执行摘要邮件，主题简洁，
重点突出：① 需要立即行动的事项 ② 最近一个月内的截止日期 ③ 违规后果
不超过300字，语气正式
```

**预期输出**：

> **主题**：【合规委〔2024〕18号】执行要求通知——请各部门负责人尽快落实
>
> 各部门负责人：
>
> **立即行动（持续性要求）**
> - 投行、研究所、资管部：涉及未公开重大信息须在24小时内登记（第3.1条）
> - 所有业务部门：单笔超50万元交易须在1个工作日内报告（第5.2条）
>
> **近期截止**：**10月31日前**全体部门完成客户信息台账梳理并报合规部备案（第2.3条）
>
> **违规后果**：超期未报的，对部门负责人进行通报批评；客户信息违规传输的，移送监管机关。
>
> 合规管理委员会
> 2024年9月10日

---

### 步骤 5：将条款转化为员工操作指引

```
第4.2条关于礼品的规定，帮我生成一个员工自查用的判断流程：
"我收到了一份礼品，我该怎么做？"用步骤图的形式
```

**预期输出**：

```
收到礼品
  ↓
估算礼品价值
  ├── 超过 500 元 → 不能接受，礼貌婉拒并说明公司规定
  └── 500 元以内 → 可以接受
        ↓
      登录礼品登记系统，填报礼品信息
        ↓
      确认年累计是否超过 2000 元
        ├── 超过 → 超出部分须上缴公司，联系合规部处理
        └── 未超过 → 登记完成，留存记录备查
```
