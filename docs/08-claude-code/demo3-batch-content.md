# 场景三：批量报告生成

> 行政部 / 运营管理部 / 各业务线综合岗 · 从原始数据到格式统一的报告

---

## 使用场景

每个季度末收集各部门汇报材料：收上来的格式不一、内容详略不齐，还要帮忙统一整理发给管理层。以下演示用 Claude Code 将结构化的原始数据批量转化为格式统一的报告，并生成管理层摘要和个性化催报邮件。

---

## 示例数据

将以下内容保存为 `部门汇报数据.md`，放在工作目录中。

??? note "展开查看示例数据"

    ```markdown
    # 各部门季度汇报原始数据

    汇报周期：2024年第三季度（7月 – 9月）

    ## 经纪业务部
    - 负责人：王志远
    - 季度目标：新开户 2000 户，佣金收入 8500 万元
    - 实际完成：新开户 2347 户（完成率 117%），佣金收入 9120 万元（完成率 107%）
    - 亮点：7月线上营销活动带来 800+ 新户，超预期
    - 问题：部分高净值客户流失，流失率 3.2%，高于上季度 1.8%
    - 下季度计划：针对高净值客户推出专属服务包，目标留存率提升至 97%

    ## 投资银行部
    - 负责人：李明华
    - 季度目标：完成 3 个 IPO 项目储备，2 个债券承销项目
    - 实际完成：IPO 储备 2 个（完成率 67%），债券承销 3 个（完成率 150%）
    - 亮点：成功完成华晨科技 5 亿元公司债承销，创部门单笔最大承销规模
    - 问题：IPO 项目受审核周期拉长影响，1 个项目推迟至 Q4
    - 下季度计划：加快推进 2 个 IPO 项目审核材料准备，新增 1 个 REITs 项目储备

    ## 资产管理部
    - 负责人：张晓敏
    - 季度目标：资管规模增长 5%（目标规模 185 亿元），新产品发行 2 只
    - 实际完成：规模达 188.5 亿元（完成率 102%），发行新产品 1 只
    - 亮点：权益类产品平均收益率 6.8%，跑赢同期沪深 300 指数 2.1 个百分点
    - 问题：固收类产品规模缩减 3.2 亿元，受市场利率波动影响
    - 下季度计划：发行 2 只混合型产品，加大对中低风险客户的产品转化

    ## 研究所
    - 负责人：陈国栋
    - 季度目标：发布深度报告 30 篇，新覆盖行业 3 个
    - 实际完成：深度报告 28 篇（完成率 93%），新覆盖行业 2 个
    - 亮点：新能源储能行业首发报告获机构客户好评，获引用 45 次
    - 问题：1 名资深分析师离职，影响电力设备板块覆盖
    - 下季度计划：招聘 2 名行业分析师，补齐电力设备和消费医疗板块

    ## 合规部
    - 负责人：刘芳
    - 季度目标：完成年度合规专项检查 Q3 部分，开展合规培训 4 次
    - 实际完成：专项检查覆盖 8 个部门（完成率 100%），培训 5 次（完成率 125%）
    - 亮点：发现并督促整改 2 起信息安全隐患，未形成监管事件
    - 问题：3 个部门合规报告提交超期
    - 下季度计划：建立合规报告自动提醒系统，推动提交率达到 100%

    ## 信息技术部
    - 负责人：赵建军
    - 季度目标：系统可用率 99.9%，完成 2 个重点系统升级
    - 实际完成：系统可用率 99.95%（超标准），完成系统升级 1 个（交易系统），另 1 个（风控系统）延期至 Q4
    - 亮点：Q3 无重大系统故障，交易高峰日承载并发量提升 40%
    - 问题：风控系统升级因第三方供应商延迟交付，推迟约 6 周
    - 下季度计划：Q4 优先完成风控系统升级，同步推进数据安全专项评估
    ```

## 操作步骤

### 步骤 1：启动 Claude Code

```bash
claude
```

### 步骤 2：批量生成 6 份部门季度报告

```
读取 部门汇报数据.md，为每个部门生成一份标准格式的季度汇报，要求：

格式模板（每份报告都要按这个结构）：
---
## [部门名称] 2024年第三季度工作汇报

**部门负责人**：[姓名]

### 一、目标完成情况
[用一段话概括完成率，重点指标加粗]

### 二、本季度亮点
[2-3条，用•列举，语气积极]

### 三、存在问题
[1-2条，客观陈述，不夸大]

### 四、下季度工作计划
[2-3条具体举措，带可量化目标]
---

请为文件中所有 6 个部门分别生成，然后保存到 quarterly_reports.md
```

**预期输出**（经纪业务部示例）：

---

**经纪业务部 2024年第三季度工作汇报**

**部门负责人**：王志远

**一、目标完成情况**

本季度经纪业务部整体超额完成目标。**新开户 2347 户**，完成率 117%；**佣金收入 9120 万元**，完成率 107%。两项核心指标均高于年初设定目标。

**二、本季度亮点**

- 7月线上营销活动成效显著，单月带来超过 800 名新客户
- 佣金收入连续两季度超额完成，同比增幅约 12%
- 数字化开户流程优化，平均开户时效缩短至 1.5 个工作日

**三、存在问题**

- 高净值客户流失率升至 3.2%，较上季度 1.8% 明显提升

**四、下季度工作计划**

- 推出高净值客户专属服务包，目标将留存率提升至 97%
- 开展客户满意度调研，Q4 覆盖率不低于 60% 的活跃客户

---

### 步骤 3：生成管理层摘要

```
基于 部门汇报数据.md，再帮我生成一份给公司管理层的季度整体摘要，要求：
1. 全公司整体完成亮点（2-3条）
2. 需要管理层关注的问题（附：提出问题的部门）
3. 下季度重点关注事项
控制在 400 字以内，语气简洁，适合在高管周会上宣读
```

**预期输出**：

> **2024年第三季度各部门工作综合摘要**
>
> **整体亮点**
> - 经纪业务、合规工作全面超额完成，信息技术系统稳定性创近年最优（可用率99.95%）
> - 资管部权益类产品收益跑赢沪深300指数2.1个百分点
> - 合规部主动发现并整改2起信息安全隐患，风险管控扎实
>
> **需关注的问题**
> - **经纪业务部**：高净值客户流失率由1.8%升至3.2%
> - **投资银行部**：IPO项目Q3完成率仅67%，Q4压力较大
> - **信息技术部**：风控系统升级因供应商延迟推迟6周
>
> **下季度重点关注**：投行IPO追赶进度、风控系统升级完成节点、高净值客户留存专项行动。

---

### 步骤 4：生成个性化催报邮件

```
假设合规部、信息技术部、研究所的报告还没交上来。
帮我给这三个部门的负责人分别写一封催报邮件，
内容要体现各部门具体情况，语气礼貌但有紧迫感，
落款：运营管理部
```

**预期输出**（研究所示例）：

> **致：陈国栋 研究所负责人**
>
> 陈所长，您好！
>
> Q3季度汇报收集工作已进入收尾阶段，目前尚未收到研究所的汇报材料。研究所本季度在新能源储能首发报告方面取得了较好成绩，也面临分析师团队调整的情况，相关内容在汇报中均可适当体现。
>
> 请于**本周五下班前**将汇报材料提交至运营管理部，如有疑问欢迎随时联系。感谢配合！
>
> 运营管理部

!!! tip "注意"
    三封邮件内容不同，每封都提到了对应部门的具体情况——这是根据原始数据个性化生成的，不是群发模板。

---

### 步骤 5（可选）：转化为 PPT 大纲

```
把季度整体摘要转换成一份 PPT 汇报的大纲，5-6 页，
每页给出：标题 + 3个要点，要点控制在 15 字以内
```
