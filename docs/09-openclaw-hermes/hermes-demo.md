# Hermes Agent 演示
### 跨会话记忆 · 操作步骤

> 演示时长约 6–8 分钟

---

## 演示核心

**"它记得你。"**

普通 AI 关掉重开就不认识你了。Hermes 有四层持久化记忆，重启后仍保留上下文。

演示分两轮：
- **第一轮**：建立记忆（介绍自己 + 说明偏好 + 记录任务）
- **第二轮**：关掉重开，验证它还记得

---

## 准备工作

- Python 3.10+
- 安装 Hermes：

```bash
pip install hermes-agent
```

- API Key（支持 Anthropic / OpenAI / DeepSeek / 本地 Ollama）

---

## 第一轮：建立记忆

### 启动

```bash
hermes
```

启动界面：

```
╔════════════════════════════════╗
║   Hermes Agent  v2.x          ║
║   Memory: 0 items             ║
╚════════════════════════════════╝

You:
```

---

### 输入 1：介绍自己

```
我是张伟，华海证券投研部的研究员，
主要覆盖新能源和储能行业。
我们部门正在准备 Q3 的行业深度报告。
```

---

### 输入 2：说明写作偏好

```
我写报告习惯先给出核心观点，
再展开数据支撑，最后说风险。
字数控制在 2000 字以内，语气专业简洁。
```

---

### 输入 3：记录任务进度

```
今天我整理了宁德时代的三季报数据，
主要关注点是储能业务毛利率的变化，
从去年的 17% 升到今年的 22%。
明天要对比比亚迪的数据。
```

**Hermes 应回复类似：**

```
我已记住你的背景和工作偏好。
宁德时代储能毛利率从 17% 升至 22% 已记录，
明天对比比亚迪时可直接引用这个基准数据。
```

---

### 退出

```
exit
```

### 验证 2：接上昨天的任务

```
昨天宁德时代的数据分析到哪了？
```

**预期回复：**

```
你昨天整理了宁德时代三季报数据，
重点是储能业务毛利率从 17% 升至 22%。
今天计划对比比亚迪的同类数据。
需要我帮你起草对比分析框架吗？
```

---

### 验证 3：按你的风格生成内容

```
帮我写一段关于储能行业的核心观点，按我的风格
```

**预期回复（自动遵循"观点→数据→风险"结构，2000 字以内）：**

```
核心观点
储能行业进入盈利拐点，头部企业毛利率持续提升，
行业竞争格局趋于集中。

数据支撑
- 宁德时代储能毛利率：17%（2022）→ 22%（2023Q3），提升 5pct
- 行业装机规模同比增速维持 60%+

风险提示
- 海外市场关税政策存在不确定性
- 原材料价格波动可能压缩利润空间
```

## 可选：查看本地记忆文件

Hermes 记忆存储在本地，不上云：

=== "Windows"

    ```cmd
    type %USERPROFILE%\.hermes\memory\long_term.json
    ```

=== "Mac / Linux"

    ```bash
    cat ~/.hermes/memory/long_term.json
    ```

文件内容示例：

```json
{
  "user_profile": {
    "name": "张伟",
    "organization": "华海证券投研部",
    "coverage": ["新能源", "储能"],
    "writing_style": "核心观点→数据→风险，2000字以内，专业简洁"
  },
  "tasks": [
    {
      "date": "2024-01-15",
      "content": "宁德时代储能毛利率 17%→22%，待对比比亚迪"
    }
  ]
}
```

!!! tip "数据不出域"
    记忆文件保存在本地机器，不会上传到任何云端，满足金融机构数据安全要求。

## 常见问题

**Q：支持哪些模型？**

支持 OpenAI、Anthropic、DeepSeek 及本地 Ollama 的 200+ 模型，换模型只需修改一个配置项。

**Q：记忆会不会越积越多？**

Hermes 有定期反思压缩机制，自动将旧记忆提炼为更精简的长期记忆，不会无限膨胀。

**Q：内存占用多少？**

相比 OpenClaw，Hermes 内存占用最高降低 72.7%，轻量部署是其设计目标之一。

**Q：OpenClaw 和 Hermes 怎么选？**

| 需求 | 推荐 |
|------|------|
| 挂载技能、扩展能力 | OpenClaw |
| 跨会话持续跟进 | Hermes |
| 两者都需要 | 可同时部署，互补使用 |
