## 角色分工

| Agent | 角色定位 | 核心职责 | 数据源 |
|-------|---------|---------|--------|
| 投研队长（主Agent） | 调度中枢 | 接收需求→拆解任务→分发→汇总快报 | 知识库问答 |
| 财务分析师 | 指标计算/估值建模 | 财务指标、利润表、ROE、毛利率、现金流 | a-stock-data + 知识库 |
| 行业分析师 | 数据采集/行业对比 | 同行对标、竞争格局、行业排名、市场份额 | a-stock-data + 数据分析 |
| 舆情专员 | 新闻/微信/舆情动态 | 最新新闻TOP5 + 深度文章TOP3 + 舆情风向 | news + wechat |
| 风控专员 | 合规扫描/风险提示 | 关联交易、解禁减持、问询函、风险信号分级 | a-stock-data + 知识库 |

## 转交规则总览

!!! danger "最容易遗漏的一步"
    **忘记配置转交描述 → 任务无法流转，主 Agent 自己回答所有问题。**

| 从 | 到 | 触发关键词 |
|----|----|-----------|
| 投研队长 | 财务分析师 | 财务指标、利润表、资产负债表、现金流、ROE、毛利率、营收、净利润、估值、PE/PB |
| 投研队长 | 行业分析师 | 行业对比、同业对标、竞争格局、市场份额、行业排名、产业链、行业趋势 |
| 投研队长 | 舆情专员 | 新闻、舆情、媒体观点、公众号文章、市场情绪、热点事件、负面报道 |
| 投研队长 | 风控专员 | 关联交易、风险提示、解禁、减持、质押、问询函、监管处罚、异常交易、合规风险 |

### 关键词互斥检查

确保以下关键词不会同时出现在两个子 Agent 的转交描述中：

| 关键词 | 归属 |
|--------|------|
| ROE/毛利率/营收/净利润 | 仅财务分析师 |
| 市场份额/行业排名 | 仅行业分析师 |
| 关联交易 | 仅风控专员（虽然涉及金额，但不归财务分析师） |
| 新闻/舆情 | 仅舆情专员 |

### 多维度同时触发

当用户提问跨多个领域时（如"分析宁德时代"），主 Agent 会同时转交多个子 Agent——这由主 Agent 提示词中的以下规则实现：

> "一个用户需求可能同时分发给多个子 Agent 并行处理"
