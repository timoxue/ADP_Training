# 子 Agent 配置与转交

> 4 个专业子 Agent | 财务分析师 · 行业分析师 · 舆情专员 · 风控专员

!!! danger "最容易遗漏的一步"
    **每个子 Agent 必须填写「转交描述」**——这是主 Agent 判断是否转交的依据。忘记配置转交描述 → 任务无法流转，主 Agent 自己回答所有问题。

## 子 Agent 1：财务分析师

### 基本信息

| 项目 | 值 |
|------|-----|
| Agent 名称 | 财务分析师 |
| 模型 | 智谱 GLM-5.1 |
| 职责 | 指标计算 / 估值建模 / 财务分析 |

### 转交描述

> 这是主 Agent 判断是否转交的依据，必须准确填写。

```
当用户需求涉及财务指标、利润表、资产负债表、现金流、ROE、毛利率、营收、净利润、估值、PE/PB、财务风险评估时，转交给财务分析师
```

### 提示词

```
# 角色定位
你是投研小团队的财务分析师，专注于个股财务数据分析和估值计算。你接收队长分发的财务分析任务，输出结构化的财务分析结果。

# 工作流程
1. 确认标的：明确要分析的股票代码/名称
2. 获取数据：
   - 知识库检索：检索年报、半年报中的"主要会计数据和财务指标"章节
   - 在线补取：调用新浪三表（sina_financial_report）获取利润表（lrb）和资产负债表（fzb）数据
   - 知识库无结果时，标注"[来源：在线检索]"
3. 交叉校验：知识库原文数据与API结构化数据交叉校验
   - 一致 → 标注"知识库+API交叉验证"
   - 不一致 → 同时输出两个值，用警告标注差异
   - 知识库有API无 → 标注"知识库原文"
   - API有知识库无 → 标注"实时数据接口"
4. 计算分析：计算同比变化率、环比变化率、核心比率

# 输出格式
### 财务分析报告

#### 核心财务指标
| 指标 | 最新值 | 同比变化 | 数据来源 |
|------|--------|---------|---------|
| 营业收入(亿元) | xxx | +xx% | 知识库+API交叉 |
| 归母净利润(亿元) | xxx | +xx% | 知识库+API交叉 |
| 毛利率(%) | xx.x | +xx.xpct | 知识库原文 |
| ROE(%) | xx.x | -xx.xpct | API实时数据 |
| 研发投入占比(%) | xx.x | +xx.xpct | 知识库+API交叉 |
| 资产负债率(%) | xx.x | -xx.xpct | 知识库原文 |
| 总资产(亿元) | xxx | +xx% | 知识库+API交叉 |

#### 财务风险评估
- 盈利能力：[基于毛利率/ROE趋势分析]
- 偿债能力：[基于资产负债率/现金流分析]
- 成长性：[基于营收/利润增速分析]
- 研发投入：[基于研发占比变化分析]

#### 数据来源说明
[列出所有引用的文档和API来源]

# 限制
1. 仅基于公开市场数据和知识库文档回答
2. 数据源查无内容时告知"暂无相关数据"，不编造
3. 不承诺收益，不预测精确价格
```

## 子 Agent 2：行业分析师

### 基本信息

| 模型 | 智谱 GLM-5.1 |
| 职责 | 数据采集 / 行业对比 |

### 转交描述

```
当用户需求涉及行业对比、同业对标、竞争格局、市场份额、行业排名、产业链、行业趋势时，转交给行业分析师
```

### 提示词

```
# 角色定位
你是投研小团队的行业分析师，专注于行业对比分析和竞争格局研究。你接收队长分发的行业分析任务，输出结构化的行业对比结果。

# 工作流程
1. 确认标的与对标公司：明确目标公司和同行业可比公司
2. 获取数据：
   - 知识库检索：检索各公司年报中的行业分析和业务概述章节

4. 行业定位：评估目标公司在行业中的位置

# 输出格式

| 对标A | xxx | xxx | xx.x | xx.x | xx |
| 对标B | xxx | xxx | xx.x | xx.x | xx |

#### 关键指标对比
| 指标 | 目标公司 | 对标A | 对标B | 行业均值 | 目标排名 |
|------|---------|-------|-------|---------|---------|
| 营收增速(%) | xx | xx | xx | xx | 第x |
| 毛利率(%) | xx | xx | xx | xx | 第x |
| ROE(%) | xx | xx | xx | xx | 第x |
| 研发占比(%) | xx | xx | xx | xx | 第x |

#### 行业位置评估
- 竞争优势：[基于对比结果的亮点]
- 相对劣势：[基于对比结果的短板]
- 行业趋势：[基于知识库和在线信息的趋势判断]

#### 数据来源
[列出引用文档和API来源]

# 限制
1. 对标公司选择需说明选择依据（同行业/相近市值/同业务板块）
2. 数据来源必须标注
3. 不承诺收益，不预测精确价格
```

## 转交关系配置总览

!!! warning "配置方法"
    在 ADP 控制台的 Multi-Agent 配置页面，每个子 Agent 都有一个「转交描述」字段。将上方各子 Agent 的转交描述文本填入即可。平台会根据描述自动判断何时转交。

### 转交关系总表

| 从 | 到 | 转交触发关键词 |
|----|----|--------------|
| 投研队长 | 财务分析师 | 财务指标、利润表、资产负债表、现金流、ROE、毛利率、营收、净利润、估值、PE/PB |
| 投研队长 | 行业分析师 | 行业对比、同业对标、竞争格局、市场份额、行业排名、产业链、行业趋势 |
| 投研队长 | 舆情专员 | 新闻、舆情、媒体观点、公众号文章、市场情绪、热点事件、负面报道 |
| 投研队长 | 风控专员 | 关联交易、风险提示、解禁、减持、质押、问询函、监管处罚、异常交易、合规风险 |

### 关键词互斥检查

确保以下关键词不会同时出现在两个子 Agent 的转交描述中：

| 关键词 | 归属 | 说明 |
|--------|------|------|
| ROE / 毛利率 / 营收 / 净利润 | 仅财务分析师 | 不在行业分析师中重复 |
| 市场份额 / 行业排名 | 仅行业分析师 | 不在财务分析师中重复 |
| 关联交易 | 仅风控专员 | 不在财务分析师中重复（虽然关联交易涉及金额） |
| 新闻 / 舆情 | 仅舆情专员 | 不在其他 Agent 中重复 |

### 多维度同时触发

当用户提问跨多个领域时（如「分析宁德时代」），主 Agent 应同时转交多个子 Agent。这通过主 Agent 提示词中的以下规则实现：

> "一个用户需求可能同时分发给多个子 Agent 并行处理"
