## 步骤 5.1：创建应用

1. 进入 ADP 平台 → 左侧「应用」→「新建应用」
2. 应用名称填写：**投研助手**
3. 选择 **Claw 模式**，点击确认

!!! danger "不可逆操作"
    Claw 模式创建后**不可切换**到其他模式。选错只能重建应用，所有配置从头来过。

## 步骤 5.2：设计提示词

框架B（Agent模式）结构如下，复制到「角色指令」文本框：

```
# 任务目标
作为投研助手，你需要帮助券商研究员和投研助理分析上市公司财务数据、
提取关联交易信息、对比关键指标，并检索相关研报和公告。

# 任务流程
1. 理解用户请求，判断是关联交易提取、指标对比，还是文档检索
2. 优先检索知识库中的年报、招股书、公告等文档
3. 调用 a-stock-data 获取结构化财务数据，与知识库数据交叉校验，不一致项⚠️标注
4. 再尝试调用实时数据接口补充
5. 按规范格式输出结构化表格，每行标注出处

# 两大产出规范
## 关联交易清单格式
| 关联方名称 | 关联关系 | 交易类别 | 交易金额 | 定价方式 | 合规依据 | 来源文档 |

## 关键指标对照表格式
| 指标 | [公司A] | [公司B] | 知识库值 | API值 | 差异 |
指标包含：营收、净利润、毛利率、净利率、ROE、资产负债率、现金流

# 知识库检索规范
- 每次回答前必须检索知识库，引用知识库原文时标注[来源：文档名 第X节]
- 知识库命中多份文档时，全部引用，不只引用第一份
- 知识库无内容时告知用户"知识库暂无相关文档"

# 限制
- 不承诺任何收益，不构成投资建议
- 知识库原文标注[来源：文档名]，API数据标注[来源：API]
- 每次回复结尾附加：本内容由AI基于知识库生成，仅供参考，不构成投资建议
```

!!! warning "第4条是基础版的局限"
    基础版第4条"再尝试调用实时数据接口补充"是模糊指令——Agent不知道该调哪个接口，大概率不触发。进阶版将这1句改为5条明确路径，见[进阶版改造指南](data-extract.md)。

## 步骤 5.3：配置知识库

### 分类结构（5个分类）

```
投研知识库/
├── 年报/          ← 上市公司年度报告（宁德时代/比亚迪/比格/宁波银行等）
├── 招股书/        ← IPO招股说明书（含关联交易章节）
├── 关联交易公告/  ← 深交所第23号格式公告
├── 问询函/        ← 交易所问询函及回复
└── 研究报告/      ← 卖方研报
```

**操作路径**：应用详情 → 知识 → 添加分类 → 上传文档

### 推荐检索配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 检索模式 | 混合检索 | 向量+关键词结合，召回更全面 |
| Top-K | 5–10 | 返回最相关的5-10段 |
| 相似度阈值 | 0.5–0.7 | 过滤低相关度结果 |
| 引用溯源 | 开启 | 显示答案来自哪个文档 |

!!! warning "等待解析完成"
    文档上传后需等待状态变为**「导入完成」**才能被检索。招股书等大文件（44万字符）解析时间较长，请耐心等待。

## 步骤 5.4：挂载 Skills（11个）

**操作路径**：Agent编辑页 → Skills → 添加 Skill

| Skills | 用途 |
|--------|------|
| `tencentcloud-adp-kb-management` | 知识库文档管理 |
| `tencentcloud-adp-kb-search` | 知识库内容检索 |
| `tencentcloud-adp-rerank` | 检索结果重排序 |
| `a-stock-data` | A股财务数据 / 巨潮公告 |
| `adp-prompt-builder` | 提示词设计辅助 |
| `adp-app-manager` | 应用创建与管理 |
| `dcf-model` | DCF估值模型 |
| `summarize-cn` | 网页/文档摘要 |
| `svg-visualizer` | 图表可视化 |
| `excel-xlsx` | Excel文件生成 |
| `word-docx` | Word文件生成 |

!!! tip "基础版不含 news 和 wechat"
    基础版没有 `news` 和 `wechat-article-search`，所以知识库外的公司无法在线补取，实时新闻也无法检索。这是基础版的已知边界，进阶版通过+2个Skill解决。

## 步骤 5.5：挂载工具

**操作路径**：Agent编辑页 → 工具 → 添加工具

- **知识库问答**：必须启用，否则Agent无法检索知识库文档
- **图文问答**（可选）：支持解读文档中的图片/表格截图

## 步骤 5.6：配置欢迎语与开场问题

**欢迎语**：
```
您好！我是投研助手，可以帮您：
1. 从年报/招股书中提取关联交易清单（含关联方/金额/合规依据）
2. 对比多家公司的关键财务指标（ROE/毛利率/现金流等），知识库与API数据交叉校验
3. 跨文档查找关联方变化（如招股书 vs 年度公告）
```

**开场问题（最多3条）**：
1. 提取宁德时代的关联交易清单
2. 对比宁德时代和比亚迪的关键财务指标
3. 宁德时代招股书和2026年预计公告里的关联方有什么变化？

## 步骤 5.7：测试调优 → 用以下3个用例验收

发布前先在「测试」页面逐一验收：

### TC-001：知识库基础检索

| 项目 | 内容 |
|------|------|
| **输入** | 提取宁德时代的关联交易清单 |
| **预期输出** | 命中4+份文档，输出结构化表格，每行标出处 |
| **通过标准** | ✅ 命中招股书+年报+预计公告+问询函；✅ 表格格式；✅ 出处标注 |

### TC-002：关键指标对照

| 项目 | 内容 |
|------|------|
| **输入** | 对比宁德时代和比亚迪的关键财务指标 |
| **预期输出** | 7项指标对照表，知识库+API交叉校验 |
| **通过标准** | ✅ 两家公司数据均提取；✅ API交叉校验；✅ 不一致项⚠️标注 |

### TC-003：跨文档交叉验证

| 项目 | 内容 |
|------|------|
| **输入** | 宁德时代招股书里的关联方，和2026年预计公告里的关联方有什么变化？ |
| **预期输出** | 招股书 vs 预计公告关联方对比表 |
| **通过标准** | ✅ 同时命中两个文档；✅ 输出对比表；✅ 消失的关联方有说明 |

TC-001~003 全部通过后，基础版完成。如需进阶版能力（在线补取/实时新闻），继续 → [进阶版改造指南](data-extract.md)
