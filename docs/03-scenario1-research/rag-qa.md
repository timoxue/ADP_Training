## 基础版 vs 进阶版操作差异

| 操作 | 基础版 | 进阶版 | 差异 |
|------|--------|--------|------|
| Skills数量 | 11个 | 13个 | +news +wechat-article-search |
| 提示词第4条 | "再尝试调用实时数据接口补充"（1句） | 5条明确补取路径（5行） | +187字符 |
| 其他步骤 | 完全相同 | 完全相同 | 无 |

!!! tip "培训要点"
    进阶版只比基础版多了2个Skill + 1条提示词改动。**最小改动，最大价值**——这就是迭代式开发的精髓：不要一次性把所有功能都加上，先跑通基础版，再针对性补能力。

## 步骤 5.1：创建应用

1. 进入 ADP 平台 → 左侧「应用」→「新建应用」
2. 选择 **Claw 模式**

!!! danger "不可逆操作"
    Claw 模式创建后**不可切换**到其他模式。选错只能重建应用，所有配置从头来过。

---

## 步骤 5.2：设计提示词

投研助手提示词使用**框架B（Agent模式）**，结构如下：

=== "基础版提示词结构"

    ```
    # 任务目标
    作为投研助手，你需要帮助券商研究员和投研助理分析上市公司财务数据、
    提取关联交易信息、对比关键指标，并检索相关研报和公告。

    # 任务流程
    1. 理解用户请求，判断是关联交易提取、指标对比，还是文档检索
    2. 优先检索知识库中的年报、招股书、公告等文档
    3. 调用 a-stock-data 获取结构化财务数据，与知识库数据交叉校验
    4. 再尝试调用实时数据接口补充（基础版：模糊指令，大概率不触发）
    5. 按规范格式输出结构化表格，每行标注出处

    # 两大产出规范
    ## 关联交易清单格式
    | 关联方名称 | 关联关系 | 交易类别 | 交易金额 | 定价方式 | 合规依据 | 来源文档 |
    
    ## 关键指标对照表格式
    | 指标 | [公司A] | [公司B] | 差异 | 备注 |
    指标包含：营收、净利润、毛利率、净利率、ROE、资产负债率、现金流

    # 限制
    - 不承诺任何收益，不构成投资建议
    - 知识库无内容时告知用户"知识库暂无相关文档"
    - 每次回复结尾附加：本内容由AI基于知识库生成，仅供参考，不构成投资建议
    - 标注数据出处，知识库原文标注[来源：文档名]
    ```

=== "进阶版第4条（关键改造）"

    将基础版第4条的1句话替换为5条明确路径：

    ```
    4. 若知识库未命中目标公司，按以下优先级在线补取：
       4.1 调用 a-stock-data 查询巨潮公告，搜索关联交易相关公告
       4.2 若公告无结果，调用 a-stock-data 查询财务数据接口
       4.3 调用 news 搜索该公司近期财经新闻（关键词：公司名+关联交易）
       4.4 调用 wechat-article-search 搜索公众号分析文章
       4.5 所有在线结果统一标注"[来源：在线检索]"，与知识库原文明显区分
    ```

    **改动量**：+187字符（从1句到5行）  
    **效果**：Agent知道确切路径，必然触发在线补取

## 步骤 5.3：配置知识库

### 知识库分类结构（5个分类）

```
投研知识库/
├── 年报/          ← 上市公司年度报告
├── 招股书/        ← IPO招股说明书
├── 关联交易公告/  ← 深交所第23号格式公告
├── 问询函/        ← 交易所问询函及回复
└── 研究报告/      ← 卖方研报
```

### 推荐配置参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 检索模式 | 混合检索 | 向量+关键词结合，召回更全面 |
| Top-K | 5–10 | 返回最相关的5-10段 |
| 相似度阈值 | 0.5–0.7 | 过滤低相关度结果 |
| 引用溯源 | 开启 | 让用户看到答案来自哪个文档 |

!!! warning "文档解析注意"
    文档上传后需等待状态变为**「导入完成」**才能被检索。招股书等大文件（44万字符）解析时间较长。

## 步骤 5.4：挂载 Skills

### 基础版（11个）

| Skills | 用途 |
|--------|------|
| tencentcloud-adp-kb-management | 知识库文档管理 |
| tencentcloud-adp-kb-search | 知识库内容检索 |
| a-stock-data | A股财务数据/巨潮公告 |
| adp-prompt-builder | 提示词设计辅助 |
| adp-app-manager | 应用管理 |
| ……（共11个） | |

### 进阶版额外增加（+2个）

| Skills | 用途 | 为什么加 |
|--------|------|---------|
| news | 财经新闻关键词搜索 | 覆盖知识库外的实时新闻 |
| wechat-article-search | 公众号文章搜索 | 补充第三方分析观点 |

!!! tip "Skills说明"
    - `news`：每次返回5条财经新闻，含标题/摘要/来源/发布时间
    - `wechat-article-search`：每次返回10篇公众号文章，含标题/概要/公众号名称

## 步骤 5.5：挂载工具

在 Agent 编辑页 → 「工具」中启用：

- **知识库问答**：让 Agent 能检索知识库文档（必须启用）
- **图文问答**（可选）：支持解读文档中的图片/表格截图

---

## 步骤 5.6：配置欢迎语与开场问题

**欢迎语示例**：
```
您好！我是投研助手，可以帮您：
1. 从年报/招股书中提取关联交易清单（含关联方/金额/合规依据）
2. 对比多家公司的关键财务指标（ROE/毛利率/现金流等）
3. 检索最新财经新闻和公众号分析文章
```

**开场问题（最多3条）**：
1. 提取宁德时代的关联交易清单
2. 对比宁德时代和比亚迪的关键财务指标
3. 宁德时代最近有什么重要公告？

## 步骤 5.7：测试调优

使用[第六步验收用例](data-extract.md)逐一测试。重点验证：

- 知识库检索是否命中4+份文档
- a-stock-data 财务数据接口是否正常调用
- 进阶版：隆基绿能等知识库外公司是否触发在线补取
- 输出格式是否符合两大产出规范
