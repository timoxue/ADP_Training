# 进阶版改造指南（在基础版基础上）

> 改动量极小：+2 个 Skills + 提示词第4条从1句改5行（+187字符）

!!! info "六步推理链导览"
    本文档是**进阶版的改造步骤**，必须先完成[基础版搭建](rag-qa.md)并通过 TC-001~003，再执行本指南。六步分析答案见 [场景一六步分析](index.md)。

## 改造步骤A：新增2个Skills

**操作路径**：Agent编辑页 → Skills → 添加 Skill

在基础版已有的11个Skills基础上，再安装：

| 新增 Skill | 能力 | 为什么加 |
|------------|------|---------|
| `news` | 财经新闻关键词搜索，每次返回5条（标题+摘要+来源+时间） | 知识库只有历史公告，news补实时动态 |
| `wechat-article-search` | 公众号文章搜索，每次返回10篇（标题+概要+公众号名称） | 补充第三方分析观点，knowledge base里没有的角度 |

!!! tip "能力边界说明"
    - `news`：无全文，SourceUrl 不可直接打开；摘要级信息，引导用户去原站查看
    - `wechat`：无全文，真实 URL 不稳定；概要级信息，标注公众号名称
    - 两者**不能替代**年报/招股书的深度提取，只能补实时动态和第三方观点

## 改造步骤B：修改提示词第4条

**操作路径**：Agent编辑页 → 角色指令

找到基础版第4条：
```
4. 再尝试调用实时数据接口补充
```

**替换为**（5条明确路径）：
```
4. 若知识库未命中目标公司，按以下优先级在线补取：
   4.1 调用 a-stock-data 查询巨潮公告，搜索关联交易相关公告
   4.2 若公告无结果，调用 a-stock-data 查询财务数据接口补充财务指标
   4.3 调用 news 搜索该公司近期财经新闻（关键词：公司名+关联交易）
   4.4 调用 wechat-article-search 搜索公众号分析文章
   4.5 所有在线结果统一标注"[来源：在线检索]"，与知识库原文明显区分
```

**改动量**：+187字符（从1句到5行）  
**为什么有效**：Agent 知道确切的调用路径和优先级，不再猜测"实时数据接口"是什么。

## 改造完成后验收：TC-004 ~ TC-006

基础版通过 TC-001~003，进阶版在此基础上需要额外通过：

### TC-004：知识库边界 — 在线补取（杀手级用例）

| 项目 | 内容 |
|------|------|
| **输入** | 提取隆基绿能的关联交易清单 |
| **基础版表现** | "知识库中暂无隆基绿能的相关文档" → 停止 |
| **进阶版预期** | 巨潮在线搜索 → 返回隆基绿能关联交易公告列表 → 标注"[来源：在线检索]" |
| **通过标准** | ✅ 在线补取成功；✅ 标注在线来源，与知识库内容区分 |

!!! warning "TC-004 是改动价值的直接证明"
    同一个问题，基础版回答"不知道"，进阶版在线搜到结果。+187字符的价值，学员一眼看见。

### TC-005：实时新闻（news Skill 专属）

| 项目 | 内容 |
|------|------|
| **输入** | 宁德时代最近有什么新闻？ |
| **基础版表现** | "知识库中暂无相关新闻" → 停止 |
| **进阶版预期** | 调用 `news` → 返回5条财经新闻（标题+摘要+来源+时间） |
| **通过标准** | ✅ 调用 news；✅ 内容是近期的；✅ 含标题/摘要/来源/时间 |

### TC-006：公众号分析文章（wechat Skill 专属）

| 项目 | 内容 |
|------|------|
| **输入** | 有没有关于比亚迪关联交易的分析文章？ |
| **基础版表现** | 知识库有公告但无分析文章 → 缺少第三方观点 |
| **进阶版预期** | 调用 `wechat-article-search` → 返回10篇公众号文章 |
| **通过标准** | ✅ 调用 wechat；✅ 知识库原文与公众号文章来源标注区分 |

## 常见坑清单（10个）

| # | 坑 | 影响 | 避坑方法 |
|---|---|------|---------|
| 1 | PDF表格解析丢结构 | 财务数据数字与字段名错位 | API交叉校验，不一致时⚠️标注 |
| 2 | mootdx TCP 7709不通 | 财务快照接口失效 | 用新浪三表HTTP接口替代 |
| 3 | 巨潮orgId错误 | 公告搜索返回0条 | 用`_cninfo_orgid()`动态查映射表 |
| 4 | 提示词整段替换 | 未先读就写，覆盖已有配置 | modify前先describe当前值 |
| 5 | 403限流 | 批量上传文档失败 | 操作间隔3秒以上 |
| 6 | news配额不足 | GetNews API调用超限 | 购买ADP套餐增购包 |
| 7 | wechat cheerio未安装 | 公众号搜索脚本报错 | `npm install -g cheerio` |
| 8 | 招股书文件过大(44万字符) | RAG分块后关联交易章节被拆碎 | 更精确提问，如"招股书第九节关联交易" |
| 9 | 新闻摘要被标注为"年报原文" | 来源混淆 | 提示词限制第9条：区分来源类型 |
| 10 | 基础版第4条"实时数据接口"太模糊 | Agent不知道该调哪个接口 | 进阶版改为5条明确路径（本文改造步骤B） |
