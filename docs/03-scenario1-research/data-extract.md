## 验收用例

| 维度 | 内容 |
|------|------|
| **输入** | 提取宁德时代的关联交易清单 |

### TC-004：知识库边界 — 基础版局限（关键测试）

| 维度 | 内容 |
|------|------|
| **输入** | 提取隆基绿能的关联交易清单 |
| **基础版预期** | "知识库中暂无隆基绿能的相关文档" → 停止 |
| **进阶版预期** | 巨潮在线搜索 → 返回隆基绿能关联交易公告列表 → 标注"[来源：在线检索]" |
| **验收标准** | 基础版：✅ 正确告知"暂无"；进阶版：✅ 在线补取成功，✅ 标注在线来源 |

!!! warning "杀手级验收用例"
    TC-004 是**杀手级验收用例**——同一个问题（隆基绿能关联交易），基础版回答"不知道"，进阶版在线搜到结果。学员一眼就能看到 +187字符提示词改动的价值。

## 常见坑清单

| # | 坑 | 影响 | 避坑方法 |
|---|---|------|---------|
| 1 | PDF表格解析丢结构 | 财务数据数字与字段名错位 | API交叉校验，不一致时⚠️标注 |
| 2 | mootdx TCP 7709不通 | 财务快照接口失效 | 用新浪三表HTTP接口替代 |
| 3 | 巨潮orgId错误 | 公告搜索返回0条 | 用`_cninfo_orgid()`动态查映射表 |
| 4 | 提示词整段替换 | 未先读就写，覆盖已有配置 | modify前先describe当前值 |
| 5 | 403限流 | 批量上传文档失败 | 操作间隔3秒以上 |
| 6 | news配额不足 | GetNews API调用超限 | 购买ADP套餐增购包 |
| 7 | wechat cheerio未安装 | 公众号搜索脚本报错 | `npm install -g cheerio` |
| 8 | 招股书文件过大(44万字符) | RAG分块后关联交易章节被拆碎 | 更精确提问，如"招股书第九节关联交易" |
| 9 | 新闻摘要被标注为"年报原文" | 来源混淆 | 提示词限制第9条：区分来源类型 |
| 10 | 基础版第4条"实时数据接口"太模糊 | Agent不知道该调哪个接口 | 进阶版改为5条明确路径 |

## 验收不过如何回溯

```
验收失败 → 判断失败类型
  ├── 知识库检索没命中 → 回第三步（检查知识库分类/文档上传状态）
  ├── API数据不准确 → 回第四步（调整a-stock-data调用参数）
  ├── 进阶版未触发在线补取 → 回第四步（检查提示词第4条5条路径是否明确）
  ├── 输出格式不符合规范 → 回第五步（调整提示词两大产出规范）
  └── Skills未被调用 → 回第五步（检查Skills挂载+工具启用状态）
```

!!! tip "培训要点"
    验收不是"测一下能跑就行"，而是要设计能暴露边界局限的用例。TC-004是最好的例子——它专门设计来暴露基础版的"在线补取缺失"这个边界。
