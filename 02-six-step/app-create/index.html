    应用创建与配置（第五步）¶

    先理解每一步做什么、为什么——再选择手工、半自动或全自动搭建。

    5A 手工操作清单¶

    □ 步骤 1：创建应用（选择 Claw 模式）
  → 在哪：ADP 平台 → 左侧「应用」→「新建应用」→ 选"Claw 模式"
  → 为什么：Skills 即装即用，灵活调用多种数据源
  → ⚠️ 不可逆：Claw 模式创建后不可切换到其他模式
  → 产出：app_id

□ 步骤 2：设计提示词
  → 在哪：Agent 编辑页 → 「角色指令」文本框
  → 为什么：提示词定义 Agent 行为逻辑（任务流程+输出要求+限制）
  → 使用框架：框架B（Agent模式）
  → 参考技能：adp-prompt-builder

□ 步骤 3：配置知识库
  → 在哪：应用详情 → 「知识」→「添加分类」→「上传文档」
  → 为什么：知识库存内部文档，供检索使用
  → 建议：混合检索、Top-K 5-10、相似度 0.5-0.7、开启引用溯源
  → 注意：文档需等待「导入完成」状态才能检索

□ 步骤 4：挂载 Skills
  → 在哪：Agent 编辑页 → 「Skills」→「添加 Skill」
  → 为什么：Claw 模式通过 Skills 调用外部数据源
  → 注意：最多 80 个 Skills

□ 步骤 5：挂载工具
  → 在哪：Agent 编辑页 → 「工具」→「添加工具」
  → 为什么：启用知识库问答、图片理解等平台内置工具

□ 步骤 6：配置平台设置
  → 在哪：应用详情 → 右上角「应用设置」
  → ⚠️ Claw 模式只有 4 项：欢迎语/对话体验/变量/高级设置(同义词)
  → ❌ 没有：内容安全/保守回答/隐私脱敏/长期记忆/上下文轮数

□ 步骤 7：配置欢迎语与开场问题
  → 在哪：Agent 编辑页 → 「欢迎语」/「开场问题」
  → 注意：开场问题最多 3 条

□ 步骤 8：测试与调优
  → 在哪：应用详情 → 「测试」→ 对话
  → 重点：知识库检索命中、Skills 调用正确、风险等级判定合理

□ 步骤 9：发布上线
  → 在哪：应用详情 → 「发布」
  → ⚠️ 高风险操作，不可撤销

    □ 步骤 1：创建应用（选择标准模式）
  → 在哪：ADP 平台 → 左侧「应用」→「新建应用」→ 选"标准模式"
  → ✅ 可切换：后续可切到单工作流/Multi-Agent
  → 产出：app_id

□ 步骤 2：配置模型
  → 在哪：Agent 编辑页 → 「模型设置」
  → 为什么：双模型协同——思考模型做推理判定，生成模型做输出

□ 步骤 3：设计提示词
  → 在哪：Agent 编辑页 → 「角色指令」文本框
  → 使用框架：框架A（标准模式），包含角色+风格+输出要求+能力限制

□ 步骤 4：配置知识库
  → 在哪：应用详情 → 「知识」
  → ✅ 标准模式知识库原生集成，比 Claw 模式的 Skill 调用更稳定精确

□ 步骤 5：配置连接器与工具
  → 在哪：Agent 编辑页 → 「连接器」→「添加连接器」
  → ⚠️ 与 Claw 模式的区别：标准模式用连接器，不是 Skills

□ 步骤 6：（可选）配置工作流
  → 在哪：Agent 编辑页 → 「工作流」→「新建工作流」
  → 适合：定期风险巡检、报告生成等固定流程

□ 步骤 7：配置平台安全设置（关键）
  → 在哪：应用详情 → 右上角「应用设置」
  → ✅ 完整设置项：内容安全 / 保守回答 / 隐私脱敏 / 长期记忆 / 上下文轮数
  → 金融场景必须开启：内容安全 + 保守回答 + 隐私脱敏

□ 步骤 8：配置欢迎语与开场问题（最多3条）

□ 步骤 9：测试与调优

□ 步骤 10：发布上线

    □ 步骤 1：创建应用（选择单工作流模式）
  → ✅ 可切换：后续可切到标准/Multi-Agent

□ 步骤 2：配置模型

□ 步骤 3：设计提示词（简短入口引导即可，核心逻辑放在工作流节点）

□ 步骤 4：配置知识库

□ 步骤 5：配置连接器与工具

□ 步骤 6：编排工作流（核心步骤）
  → 在哪：Agent 编辑页 → 「工作流」→「新建工作流」→ 拖拽节点
  → 常用节点：
    开始节点：接收用户输入
    知识检索节点：从知识库检索文档
    大模型节点：分析/摘要/分类
    条件判断节点：按结果走不同分支（详见场景三方案A）
    代码节点：运行代码处理数据
    文件收集节点：收集用户上传文件
    迭代节点：遍历文件列表逐一处理
    结束节点：返回最终结果
  → 参考技能：adp-workflow-package-generator

□ 步骤 7：配置平台安全设置
  → ✅ 单工作流有完整设置项（同标准模式）

□ 步骤 8：配置欢迎语（最多3条开场问题）

□ 步骤 9：测试工作流（重点检查节点执行顺序、条件分支触发）

□ 步骤 10：发布上线

    □ 步骤 1：创建应用（选择 Multi-Agent 模式）
  → ✅ 可切换：后续可切到标准/单工作流

□ 步骤 2：配置模型（调度Agent用强推理模型，子Agent用生成模型）

□ 步骤 3：定义子Agent结构（先规划，再配置）
  ┌─────────────────────────────────────────────┐
  │ 调度Agent：接收问题→判断类型→分发→汇总         │
  │   转交条件：                                   │
  │     - 涉及[能力1] → 转交 [子Agent1]            │
  │     - 涉及[能力2] → 转交 [子Agent2]            │
  ├─────────────────────────────────────────────┤
  │ 子Agent1：[名称]                              │
  │   职责 / 专用技能 / 输出格式 / 转交描述          │
  │ 子Agent2：...                                 │
  └─────────────────────────────────────────────┘
  → ⚠️ 子Agent数量建议不超过5个

□ 步骤 4：编写每个Agent的提示词
  → 调度Agent：任务目标+转交规则+汇总模板
  → 子Agent：专项任务+标准化输出格式+边界限制

□ 步骤 5：配置子Agent转交描述
  → 一句话描述"什么情况下转交给这个子Agent"
  → ⚠️ 转交描述必须互斥，避免重叠导致调度混乱

□ 步骤 6：配置知识库
  → ⚠️ Multi-Agent 模式下需手动启用"知识库问答"工具

□ 步骤 7：挂载技能（应用级别，所有子Agent共享）
  → 通过提示词约束各子Agent只用自己领域的数据源

□ 步骤 8：配置平台安全设置
  → ✅ Multi-Agent 有完整设置项（同标准模式）

□ 步骤 9：配置欢迎语（最多3条开场问题）

□ 步骤 10：测试（重点：调度准确性、子Agent输出格式、汇总完整性）

□ 步骤 11：发布上线

    5B 搭建方式选择¶

    展示完手工操作清单、用户确认理解后，再询问搭建方式：

    方式 说明 适合谁
    手工搭建 用户自己在 ADP 控制台按步骤操作 想完全掌控每步、学习操作流程
    半自动搭建 Claude 完成可自动化步骤（创建应用、提示词、知识库分类、挂载Skills、欢迎语），用户手动完成平台设置 想省时间但想亲手做关键设置
    全自动搭建 Claude 完成所有可自动化步骤，最后给出手动操作清单 想最快交付，理解了原理但不想动手

    自动搭建前必须二次确认

    执行 create_app 之前，必须向用户展示：应用名称、应用模式、提示词摘要、知识库结构、Skills 清单。用户明确回复"是/确认/OK"后才执行。模糊回复不算确认。

    自动搭建调用技能¶

    步骤 调用技能 适用模式
    创建应用 adp-app-manager 全部
    设计提示词 adp-prompt-builder → adp-app-manager 全部
    Multi-Agent 子Agent提示词 adp-prompt-builder + public-adp-multiagent-creator Multi-Agent
    配置知识库 tencentcloud-adp-kb-management 全部
    挂载 Skills/工具 adp-app-manager 全部
    配置欢迎语 adp-app-manager 全部
    配置工作流 adp-workflow-package-generator 标准/单工作流
    配置连接器 adp-plugin-manager 全部
    平台安全设置 ❌ 无法自动 标准/单工作流/Multi-Agent
    发布上线 adp-app-manager（二次确认） 全部

    平台安全设置必须手动开启

    内容安全、保守回答、隐私脱敏、长期记忆等设置项无法通过 API 自动配置，必须在 ADP 控制台手动开启。这是自动搭建流程的必要手动步骤。

    知识库配置要点¶

    参数 推荐值 说明
    检索模式 混合检索 向量+关键词结合，召回更全面
    Top-K 5–10 返回最相关的5-10段
    相似度阈值 0.5–0.7 过滤低相关度结果
    引用溯源 开启 让用户看到答案来自哪个文档
    文档状态 导入完成 等待后才能检索

    发布前检查清单¶

    • [ ] 提示词已配置（核心逻辑已定义）
    • [ ] 知识库文档状态为"导入完成"
    • [ ] Skills/工具/连接器已挂载
    • [ ] 欢迎语已配置（开场问题≤3条）
    • [ ] 平台安全设置已开启（金融场景：内容安全+保守回答+隐私脱敏）
    • [ ] 测试用例已通过（高风险/低风险/边界case）
    • [ ] 免责声明已嵌入提示词（金融场景必须）
