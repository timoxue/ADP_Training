# Hermes Agent 演示脚本
### 跨会话记忆 · 现场操作步骤

> 预计演示时长：6–8 分钟  
> 适合手机阅读，步骤逐条执行

---

## 演示核心

Hermes 最打动人的能力只有一句话：

**"它记得你。"**

演示分两轮：
- 第一轮：和 Hermes 说话，建立记忆
- 第二轮：关掉重开，它还记得

---

## 演示前提

- 已安装 Python 3.10+
- 已安装 Hermes：

```bash
pip install hermes-agent
```

- 有 API Key（支持 Anthropic / OpenAI / DeepSeek）

---

## 第一步：启动 Hermes（第一轮）

```bash
hermes
```

看到以下界面说明启动成功：

```
╔════════════════════════════════╗
║   Hermes Agent  v2.x          ║
║   Memory: 0 items             ║
╚════════════════════════════════╝

You:
```

---

## 第二步：建立记忆（输入以下内容）

**第一条：介绍自己**

```
我是张伟，华海证券投研部的研究员，
主要覆盖新能源和储能行业。
我们部门正在准备 Q3 的行业深度报告。
```

按回车，等 Hermes 回复。

---

**第二条：说明工作偏好**

```
我写报告习惯先给出核心观点，
再展开数据支撑，最后说风险。
字数控制在 2000 字以内，语气专业简洁。
```

按回车，等 Hermes 回复。

---

**第三条：记录一个任务进度**

```
今天我整理了宁德时代的三季报数据，
主要关注点是储能业务毛利率的变化，
从去年的 17% 升到今年的 22%。
明天要对比比亚迪的数据。
```

按回车，等 Hermes 回复。

**Hermes 应该会说类似：**

```
我已经记住了你的背景和工作风格。
宁德时代储能毛利率从17%升至22%，
这个数据我会保留，明天你继续分析比亚迪时可以直接引用。
```

---

## 第三步：退出第一轮

```
exit
```

或按 `Ctrl+C`

---

## 第四步：重新启动 Hermes（第二轮）

```bash
hermes
```

注意看启动信息变化了：

```
╔════════════════════════════════╗
║   Hermes Agent  v2.x          ║
║   Memory: 3 items             ║  ← 这里从 0 变成了 3
╚════════════════════════════════╝

You:
```

## 第五步：验证记忆（让观众看到效果）

**测试 1：问它你是谁**

```
我是谁，你还记得吗？
```

**预期回复：**

```
你是张伟，华海证券投研部研究员，
主要覆盖新能源和储能行业，
正在准备 Q3 行业深度报告。
```

---

**测试 2：直接接上昨天的任务**

```
昨天宁德时代的数据分析到哪了？
```

**预期回复：**

```
你昨天整理了宁德时代三季报数据，
重点是储能业务毛利率从 17% 升至 22%。
今天计划对比比亚迪的同类数据。
需要我帮你起草对比分析框架吗？
```

---

**测试 3：让它按你的风格写一段**

```
帮我写一段关于储能行业的核心观点，按我的风格
```

**预期回复（按"核心观点→数据→风险"结构，2000字以内）：**

```
**核心观点**
储能行业进入盈利拐点，头部企业毛利率持续提升，
行业竞争格局趋于集中。

**数据支撑**
- 宁德时代储能毛利率：17%（2022）→ 22%（2023Q3），提升 5pct
- 行业装机规模同比增速维持 60%+
- ...

**风险提示**
- 海外市场关税政策存在不确定性
- 原材料价格波动可能压缩利润空间
```

## 第六步：展示记忆文件（可选加分项）

Hermes 的记忆存在本地文件里，可以直接打开：

```bash
# Windows
type %USERPROFILE%\.hermes\memory\long_term.json

# Mac / Linux
cat ~/.hermes/memory/long_term.json
```

**看到的内容类似：**

```json
{
  "user_profile": {
    "name": "张伟",
    "organization": "华海证券投研部",
    "coverage": ["新能源", "储能"],
    "writing_style": "核心观点→数据→风险，2000字以内，专业简洁"
  },
  "tasks": [
    {
      "date": "2024-01-15",
      "content": "宁德时代储能毛利率 17%→22%，待对比比亚迪"
    }
  ]
}
```

**讲师话术：**
> "大家看到这个 JSON 文件，这就是 Hermes 的长期记忆。
> 它不在云端，就在你的本地机器上。
> 如果你们有数据不出域的要求，Hermes 完全满足。"

## 讲师话术提示

**开场（30 秒）：**
> "现在来看 Hermes。
> 我之前说 Hermes 是'体验派'——
> 核心卖点就四个字：跨会话记忆。
> 普通 AI 是金鱼记忆，你关掉重开，它就不认识你了。
> Hermes 不一样，我现在演示给大家看。"

**第一轮结束后（10 秒）：**
> "好，我现在关掉它，重新打开，
> 看看它还认不认识张伟。"

**记忆验证亮点（20 秒）：**
> "你们注意启动界面，
> 第一次是 Memory: 0，
> 现在是 Memory: 3。
> 它记住了三件事：你是谁、你怎么写报告、昨天工作到哪了。"

**结尾对比（20 秒）：**
> "OpenClaw 和 Hermes 的区别在这里：
> OpenClaw 擅长'挂技能'，让 Agent 会做更多事；
> Hermes 擅长'存记忆'，让 Agent 越用越懂你。
> 两个框架解决的是不同的问题，不是竞争关系。"

## 常见问题

**Q：内存占用多少？**

> Hermes 比 OpenClaw 最高低 72.7%，
> 轻量是它的设计目标。

**Q：支持哪些模型？**

> 支持 OpenAI、Anthropic、DeepSeek，
> 以及本地 Ollama 的 200+ 模型，
> 换模型只改一个配置项。

**Q：记忆会不会越积越多？**

> Hermes 有定期反思压缩机制，
> 自动把旧记忆提炼成更精简的长期记忆，
> 不会无限膨胀。
