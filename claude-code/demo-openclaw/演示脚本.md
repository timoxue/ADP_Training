# OpenClaw 演示脚本
### 合规预检技能 · 现场操作步骤

> 预计演示时长：8–10 分钟  
> 适合手机阅读，步骤逐条执行

---

## 演示前提

- 已安装 Node.js 18+
- 已安装 OpenClaw（`npm install -g openclaw`）
- 本地有这个目录：`claude-code/demo-openclaw/`

---

## 第一步：放入技能文件

技能文件已经在：

```
claude-code/demo-openclaw/skills/
  securities-compliance-check/
    SKILL.md
```

把整个 `skills/` 目录复制到 OpenClaw 技能目录：

```bash
# Windows
xcopy /E /I skills "%USERPROFILE%\.openclaw\skills"

# Mac / Linux
cp -r skills ~/.openclaw/skills
```

验证文件在位：

```bash
ls ~/.openclaw/skills/securities-compliance-check/
# 应该看到 SKILL.md
```

---

## 第二步：启动 OpenClaw Gateway

```bash
openclaw start
```

看到以下输出说明启动成功：

```
✓ OpenClaw Gateway running on port 18789
✓ Skills loaded: securities-compliance-check
✓ Ready
```

---

## 第三步：打开对话界面

方式一（浏览器）：
```
http://localhost:18789
```

方式二（命令行）：
```bash
openclaw chat
```

---

## 第四步：演示合规预检

### 场景 A：检查一段营销话术

把下面这段话粘贴进对话框：

---

**输入：**

```
请帮我做合规预检：

"张先生您好，我们这款产品历史年化收益稳定在 8% 以上，
基本上是保本的，买了肯定不会亏。
您把账号密码告诉我，我帮您操作，
保证收益比银行理财高。"
```

---

**预期输出：**

```
## 合规预检报告

**检查对象**：营销话术
**整体风险等级**：🔴 高

### 风险条目

| 序号 | 原文摘录 | 风险类型 | 风险等级 | 整改建议 |
|------|---------|---------|---------|---------|
| 1 | 年化收益稳定在8%以上 | 收益承诺 | 🔴 高 | 改为"历史业绩不代表未来收益" |
| 2 | 基本上是保本的 | 收益承诺 | 🔴 高 | 删除，不得承诺本金安全 |
| 3 | 买了肯定不会亏 | 收益承诺 | 🔴 高 | 删除 |
| 4 | 您把账号密码告诉我 | 代客操作 | 🔴 高 | 删除，绝对禁止 |
| 5 | 我帮您操作 | 代客操作 | 🔴 高 | 删除，绝对禁止 |
| 6 | 保证收益 | 收益承诺 | 🔴 高 | 删除 |

### 合规结论
❌ 禁止使用

### 整改版本
"张先生您好，我们这款产品历史业绩表现较好，
但过往收益不代表未来表现，投资有风险。
如需了解详情，建议您联系持牌投资顾问进行专业评估。"

---
⚠️ 本报告由 AI 辅助生成，仅供内部参考，最终合规判断以合规部意见为准。
```

---

### 场景 B：检查一段相对合规的话术

**输入：**

```
请帮我做合规预检：

"李女士，根据您的风险评估结果，
您属于平衡型投资者。
这款产品风险等级为 R3，与您的风险承受能力匹配。
历史年化收益约 6%，但过往业绩不代表未来表现，
投资有风险，请您慎重考虑。"
```

**预期输出：**

```
## 合规预检报告

**检查对象**：营销话术
**整体风险等级**：🟢 低

### 风险条目

| 序号 | 原文摘录 | 风险类型 | 风险等级 | 整改建议 |
|------|---------|---------|---------|---------|
| 1 | 历史年化收益约 6% | 表述不规范 | 🟡 中 | 建议加注"该数据来源于XXX区间，仅供参考" |

### 合规结论
✅ 通过（建议补充数据来源标注后使用）

---
⚠️ 本报告由 AI 辅助生成，仅供内部参考，最终合规判断以合规部意见为准。
```

## 第五步：展示技能结构（可选加分项）

现场打开 SKILL.md 给观众看：

```bash
cat ~/.openclaw/skills/securities-compliance-check/SKILL.md
```

**讲解要点：**

> "大家看到的这个文件，就是这个技能的全部内容。
> 只有 80 行左右，一个合规专员花半小时就能写完。
> 写完放进目录，重启 Gateway，技能立刻生效。
> 这就是 OpenClaw 技能体系的核心价值——
> 能力可以沉淀，可以复用，可以在团队里共享。"

---

## 第六步：关闭演示

```bash
openclaw stop
```

## 讲师话术提示

**开场（30 秒）：**
> "接下来我们来看 OpenClaw。
> 我刚才说，养一个'虾'分三步：定人格、立规矩、建能力。
> 我们现在跳到第三步，直接演示怎么给这只虾装上一个合规预检的技能。"

**技能创作亮点（30 秒）：**
> "这个技能不是我从网上下载的——
> 市面上确实没有专门针对证券公司的合规技能。
> 这是我昨晚用 Claude Code 写的，花了不到 10 分钟。
> 这说明什么？技能不依赖第三方，你的业务需求自己写，自己用，数据不出域。"

**红线强调（20 秒）：**
> "大家注意看这三条红线——
> 不承诺收益、不代客操作、不构成投资建议。
> 这三条不是我发明的，是证券监管的基本要求。
> 现在我们把它固化进了 AI 的行为里。"
