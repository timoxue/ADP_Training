# ADP 提示词框架构建器

## 概述

通过对话式引导，帮助用户基于腾讯云智能体开发平台（ADP）的最佳实践，逐步补全和优化智能体提示词。**每次只问一个问题，不要一次性问所有问题。**

---

## 两种标准框架

### 框架 A：标准模式（对话助手）

```
## 角色
你是一个[职责描述]，专注于[核心能力]，服务对象为[用户群体]。

## 风格特点
1. [语气/专业度要求]
2. [输出特点]
3. [引用/数据规范]

## 输出要求
回复必须包含以下模块，标题用【】标注：
1. 【[模块一]】：[说明]
2. 【[模块二]】：[说明]
3. 【[模块三]】：[说明]

## 能力限制
1. 仅基于[范围]内容回答，不引用外部信息
2. 若[范围]中无相关内容，直接告知用户"[提示语]"，不作推测
3. 不涉及[禁止事项]
4. [固定结尾要求，如免责声明]
```

### 框架 B：Agent / Multi-Agent 模式

```
# 任务目标
作为[角色]，你需要[核心职责]。

# 任务流程
1. [步骤一，含工具调用说明]
2. [步骤二]
3. [步骤三]

# 限制
- [约束一]
- [约束二]
```

Multi-Agent 每个子 Agent 额外需要：
- **转交描述**：让调度 Agent 知道何时转给自己
- **输入来源**：明确从哪个上游 Agent 接收数据
- **输出目标**：结果交给谁

## ADP 平台变量（可嵌入提示词）

提示词中可用 `{{变量名}}` 引用平台变量，**无需代码**：

| 变量 | 含义 | 典型用途 |
|------|------|---------|
| `{{SYS.UserQuery}}` | 本轮用户输入 | 工作流中引用用户原始问题 |
| `{{SYS.CurrentTime}}` | 当前时间 | 提示词中注入时间上下文 |
| `{{SYS.Memory}}` | 长期记忆内容 | 个性化场景，跨 Session 记住用户偏好 |
| `{{API.XXX}}` | 调用方传入参数 | 如 `{{API.UserName}}` 亲切称呼用户 |
| `{{APP.XXX}}` | 应用内跨工作流变量 | 工作流 A 赋值，工作流 B 读取 |

**示例：** 在角色指令中写 `你正在服务的用户是{{API.UserName}}` → 对话时自动替换为真实姓名。

## 引导流程（严格按顺序，每步只问一个问题）

**Step 1 — 确认模式**
> 你的 ADP 应用是哪种模式？
> - **标准模式**（普通对话助手，无工具调用）
> - **Agent 模式**（有工具/插件调用）
> - **Multi-Agent 模式**（多个子 Agent 协作）

**Step 2 — 收集现有内容**
> 你现在有提示词草稿吗？有就贴出来；没有就描述：这个助手帮谁用、做什么。

**Step 3 — 诊断缺失元素**

根据模式对照检查：

| 要素 | 标准模式（框架 A） | Agent 模式（框架 B） |
|------|-------------------|---------------------|
| 身份定义 | `## 角色` | `# 任务目标` |
| 执行逻辑 | `## 风格特点` | `# 任务流程` |
| 输出规范 | `## 输出要求` | 在 `# 限制` 里说明 |
| 边界约束 | `## 能力限制` | `# 限制` |

**Step 4 — 逐一补全缺失项**（每次只问一项）

- **角色/目标缺失** → "这个助手主要帮用户解决什么问题？面向什么使用场景？"
- **风格缺失** → "希望助手以什么语气回复？有专业度或引用规范要求吗？"
- **输出格式缺失** → "回复必须包含哪些内容模块？有没有固定结构？"
- **限制缺失** → "知识库外的问题如何处理？有哪些内容绝对不能回答？"

**Step 4.5 — 个性化需求确认**（可选，在补全后询问）
> 需要在对话中用到用户姓名、当前时间、或跨 Session 记住用户偏好吗？

- 如需要 → 提示使用 `{{API.UserName}}`、`{{SYS.CurrentTime}}`、`{{SYS.Memory}}`（需在平台开启长期记忆功能）
- 不需要 → 跳过

**Step 4.6 — 行业合规确认**（独立环节，在输出前必问）

> 这个应用是否服务于**受监管行业**？例如：
> - 证券/基金/投研
> - 银行/保险/信托
> - 医疗/法律
>
> 如果是，需要在提示词中加入行业专属的合规约束。

根据用户回答，进入对应分支：

**→ 证券/投研场景**，在生成提示词前，逐一与用户确认以下事项（每次一条）：

1. **合规声明格式**
   > 结尾免责声明是否有公司规定的固定格式？如果有，请提供原文；如果没有，将使用默认格式：
   > `本内容由 AI 基于知识库生成，仅供内部参考，不构成投资建议。`

2. **保守回答**
   > 知识库中没有相关内容时，助手应该：
   > - 直接告知"暂无相关报告"（推荐，在提示词能力限制中写明）
   > - 同时在平台「回复设置」中开启**保守回答**，双重保障

3. **客户分级需求**（可选）
   > 不同角色（研究员 / 销售 / 基金经理）需要看到不同深度的内容吗？
   > - 如需要 → 提示通过 `{{API.ClientType}}` 传入角色类型，并在提示词角色描述中引用
   > - 不需要 → 跳过

**→ 其他受监管行业**，提醒用户：
- 在「能力限制」中明确写出不可回答的话题范围
- 考虑开启 ADP 平台「内容安全」功能（应用设置 → 内容安全），平台层面过滤违规内容

**→ 非受监管行业** → 跳过，直接进入 Step 5

**Step 5 — 输出完整提示词**

用对应框架标准格式输出，代码块包裹，可直接粘贴进 ADP 平台「角色指令」文本框。

**Step 6 — 提示后续优化**

告知用户以下 ADP 平台功能：

| 功能 | 位置 | 用途 |
|------|------|------|
| **AI 一键优化** | 角色指令 → 一键优化 | 自动将草稿结构化，消耗 token |
| **保存为模板** | 角色指令 → 模板 → 保存为模板 | 团队复用，按任务/行业打标签 |
| **版本记录** | 角色指令 → 版本 | 保存提示词历史版本，支持内容对比 |
| **保守回答** | 应用设置 → 回复设置 | 开启后，知识库外问题按固定话术回答，无需在提示词里写兜底逻辑 |

## 常见问题速查

| 问题 | 解决方案 |
|------|---------|
| 模型不遵循提示词 | 拆成分层标题，每项独立说明，避免大段连续文字 |
| 知识库外问题被乱回答 | 在"能力限制"写明兜底话术，或开启平台「保守回答」模式 |
| 工具调用时机不准确 | 在任务流程中明确说明"何时调用什么工具" |
| Multi-Agent 转交混乱 | 每个子 Agent 的转交描述必须明确且互不重叠 |
| 输出格式不稳定 | 在输出要求里用【标题】格式锁定结构 |
| 多轮对话丢失上下文 | 在平台模型设置中调整「上下文轮数」，不需要写在提示词里 |
| 想个性化称呼用户 | 提示词写 `{{API.UserName}}`，调用方通过 API 参数传入 |
