# 各步骤填写模板 + 典型场景模板

本文件提供六步推理链各步骤的填写模板，以及常见场景的快速起步模板。

---

## 交付物定义模板（第一步）

```
应用名称：[XX 助手]
目标用户：[谁在用——角色/岗位/特征]
核心场景：[什么情况下用——触发条件/使用时机]
交付物形态：[用户拿到什么——报告/清单/建议/答案/推荐/...]
成功标准：[怎么算做成了——1-3 条可验证的标准]
```

---

## 能力拆解模板（第二步）

```
交付物：[从第一步]

能力清单：
  1. [能力名称]（P0/P1/P2）：[具体描述——需要什么、做到什么程度]
  2. [能力名称]（P0/P1/P2）：[具体描述]
  3. [能力名称]（P0/P1/P2）：[具体描述]
  ...

优先级说明：
  P0 = 必须有，缺了应用不可用
  P1 = 锦上添花，有更好
  P2 = 未来扩展，本期不做
```

---

## 功能映射模板（第三步）

```
能力1：[XXX] → 平台功能：[YYY] → 技能：[ZZZ]
  - 配置详情：[具体怎么配]
  - 注意事项：[已知限制]

能力2：[XXX] → 平台功能：[YYY] → 技能：[ZZZ]
  - 配置详情：[具体怎么配]
  - 注意事项：[已知限制]
...
```

## 设计推理模板（第四步）

```
决策：[简述决策内容]
选它：[为什么选这个方案]
不选：[考虑过但放弃的替代方案，为什么放弃]
边界：[这个方案做到什么程度就到极限了]
兜底：[超出边界时怎么处理]
```


## 典型场景快速起步模板

### 场景 A：知识库问答助手

```
应用名称：[领域]问答助手
目标用户：[领域]从业者/客户
核心场景：查询[领域]相关问题
交付物：结构化答案+来源引用
能力：知识检索(P0)+内容生成(P1)
映射：知识库→tencentcloud-adp-kb-management + 提示词→adp-prompt-builder
关键决策：纯知识库模式（无实时数据需求）
```

### 场景 B：数据分析助手

```
应用名称：[业务]分析助手
目标用户：分析师/管理层
核心场景：查看数据、生成图表、分析趋势
交付物：数据图表+分析结论+建议
能力：数据查询(P0)+推理分析(P0)+内容生成(P1)+图表生成(P1)
映射：a-stock-data/数据源 + data-analysis-workflow + 提示词
关键决策：数据源选型（API vs 知识库 vs 两者结合）
```

### 场景 C：客服/咨询助手

```
应用名称：[产品]客服助手
目标用户：终端客户
核心场景：产品咨询、问题解答、投诉处理
交付物：问题解答+操作指引+转人工提示
能力：知识检索(P0)+推理分析(P1)+合规风控(P0)+记忆(P1)
映射：知识库→tencentcloud-adp-kb-management + 提示词(合规约束) + 长期记忆
关键决策：合规约束层级（提示词+平台安全），人工兜底机制
```

### 场景 D：文档生成助手

```
应用名称：[业务]文档助手
目标用户：需要定期产出文档的人员
核心场景：按模板生成报告/合同/方案
交付物：Word/Excel/PPT/PDF 文档
能力：知识检索(P1)+内容生成(P0)+文档生成(P0)
映射：知识库 + 提示词 + word-docx/excel-xlsx/powerpoint-pptx/pdf
关键决策：文档模板设计（固定模板 vs 灵活生成）
```

### 场景 E：多步骤流程助手

```
应用名称：[业务]流程助手
目标用户：需要走固定流程的人员
核心场景：信息采集→审核→执行→反馈
交付物：流程执行结果+状态通知
能力：内容生成(P0)+流程编排(P0)+外部集成(P1)
映射：提示词 + adp-workflow-package-generator + adp-plugin-manager
关键决策：标准模式 vs 工作流模式（多步骤建议工作流）
```
