    财富顾问助手详细搭建指南（第五步详细操作）¶

    标准模式 | 5分类知识库 | 提示词含四条红线+三模块输出

    六步推理链导览

    本文档是第五步：分步操作的完整配置指南。六步分析答案见 场景二六步分析。

    操作清单（9步）¶

    步骤 做什么 在哪操作 为什么
    5.1 创建应用 新建应用，选择标准模式 ADP → 应用 → 新建应用 → 标准模式 知识库原生集成更稳定，支持长期记忆
    5.2 配置模型 选择思考模型+生成模型 应用编辑页 → 模型设置 双模型协同，推理+生成分工
    5.3 设计提示词 粘贴角色指令（含四条红线） 应用编辑页 → 角色指令 定义Agent行为边界和合规约束
    5.4 配置知识库 创建5分类 → 上传6份文档 应用详情 → 知识 产品数据的唯一来源
    5.5 配置变量与记忆 添加3个变量+开启长期记忆 应用设置 → 变量与记忆 客户个性化+跨会话记忆
    5.6 配置欢迎语 设置欢迎语+3条开场问题 应用编辑页 → 欢迎语 引导用户快速开始，验证红线二
    5.7 测试与调优 用CP/KB/M系列用例验收 应用详情 → 测试 确保提示词/知识库协同正常
    5.8 发布上线 发布到正式环境 应用详情 → 发布 发布后才能正式使用
    5.9 切换CRM传参 删除默认值，改为API传参 应用设置 → 变量与记忆 上线阶段接入真实客户数据

    步骤 5.1：创建应用（标准模式）¶

    这是最关键的一步——选错模式要从头来过！

    1. 登录 ADP 控制台
    2. 左侧导航 → 应用
    3. 点击 新建应用
    4. 填写：
    5. 应用名称：财富顾问助手标准版
    6. 应用描述：银行理财客户专属投顾助手，支持产品查询、费率对比、风险匹配、合规问答
    7. 选择模式：标准模式（不是 Claw 模式！）
    8. 点击确认创建
    项 说明
    ⚠️ 模式不可逆 Claw 模式创建后不可切换到其他模式；标准模式可切到单工作流/Multi-Agent
    ✅ 标准模式优势 知识库原生集成更稳定；双模型协同推理；支持长期记忆；未来可切工作流扩展

    产出 app_id：2068970251895277568

    步骤 5.2：配置模型¶

    1. 进入应用编辑页
    2. 左侧菜单 → 模型设置
    3. 选择模型：DeepSeek V3.2（思考模型和生成模型均选此项）
    4. 保存

    标准模式采用双模型协同： - 思考模型：做推理判定（风险等级匹配、合规检查） - 生成模型：做输出（格式化回复、产品推荐报告）

    步骤 5.3：设计并粘贴提示词¶

    1. 左侧菜单 → 角色指令
    2. 将以下提示词完整粘贴到文本框中
    3. 保存
    ## 角色
你是{{API.CustomerName}}的专属财富顾问助手，服务于[公司名]财富管理部，客户风险等级为{{API.RiskLevel}}。

以下是你对该客户的历史了解：
{{SYS.Memory}}

## 风格特点
1. 亲切、简洁，避免专业术语堆砌
2. 每次回答费率、投资范围问题时必须引用知识库原文并注明出处
3. 主动记住客户提及的偏好和关切，下次对话中体现

## 记忆要求
每次对话后，请自动记录以下客户信息更新：
- 客户明确表达的风险偏好
- 持仓产品或关注的产品类型
- 客户提出的服务诉求或投诉
- 客户家庭/职业背景（客户主动提及时）

## 输出要求
回复必须包含以下模块，标题用【】标注：
1. 【直接回答】：针对客户问题给出简明答案
2. 【产品依据】：引用知识库，标注来源文件
3. 【风险提示】：每条回复末尾必须附加一句风险提示

## 四条红线（绝对禁止，无例外）

**红线一：不承诺收益**
禁止出现"预期收益""历史收益即未来收益""一定能赚"等表述。
遇到客户问"能赚多少"→ 回答："收益因市场情况而异，过往业绩不代表未来表现，请参阅产品说明书风险揭示章节。"

**红线二：不输出"保本""无风险""稳赚"**
一律替换为："该产品存在本金损失风险，请根据您的风险承受能力谨慎选择。"

**红线三：不代客操作**
不接收"帮我买XXX""帮我赎回"等指令。
遇到此类请求 → 回答："操作需通过官方App/柜台完成，我无法代您进行交易。"

**红线四：超出知识库范围不作答**
若产品信息不在知识库中 → 回答："该产品信息暂未收录，建议联系您的持牌投顾[联系方式]确认。"

## 能力限制
1. 仅基于知识库中的产品资料作答
2. 所有回答不构成投资建议，最终决策须客户自主作出
3. 不涉及客户账户余额、持仓数据、交易记录（此类信息走核心系统接口，不经过本助手）

    提示词结构说明¶

    模块 作用 对应能力
    角色 定义身份+注入客户信息 个性化
    风格特点 定义回答风格+知识库引用规范 内容生成
    记忆要求 定义跨会话记忆内容 记忆个性化
    输出要求 定义三模块输出格式 内容生成
    四条红线 合规硬约束 合规风控
    能力限制 明确边界 兜底

    变量说明¶

    变量 来源 替换时机
    {{API.CustomerName}} CRM 系统通过 custom_vars 传入 每次对话开始时
    {{API.RiskLevel}} CRM 系统通过 custom_vars 传入 每次对话开始时
    {{API.CustomerID}} CRM 系统通过 custom_vars 传入 用于记忆隔离
    {{SYS.Memory}} ADP 平台长期记忆自动注入 每次对话开始时

    步骤 5.4：配置知识库¶

    分类结构（5个分类）¶

    操作路径：左侧菜单 → 知识 → 添加分类 → 上传文档

    知识库/
├── 基金理财产品说明书/  ← 基金招募说明书PDF
├── 费率表/              ← 产品费率对照表
├── 适当性匹配规则/      ← 风险等级匹配规则
├── 常见问题FAQ/         ← 客户常见问题及标准回答
└── 产品禁售限售名单/    ← 禁售/限售产品清单

    文档清单（6份）¶

    分类 文档 格式 大小
    基金理财产品说明书 基金招募说明书1.pdf PDF 1.1MB
    基金理财产品说明书 基金招募说明书2.pdf PDF 565KB
    费率表 基金费率表.xlsx XLSX 7KB
    适当性匹配规则 适当性匹配规则.md MD 2KB
    常见问题FAQ 常见问题FAQ.md MD 4KB
    产品禁售限售名单 产品禁售限售名单.md MD 2KB

    等待解析完成

    上传后需等待文档解析完成：解析中 → 审核中 → 学习中 → 导入完成。 全部文档显示"导入完成"后才能测试知识库检索！

    检索配置建议¶

    参数 建议值 说明
    检索方式 混合检索 向量+关键词，兼顾语义和精确匹配
    Top-K 5-10 返回最相关的5-10个片段
    相似度阈值 0.5-0.7 太低噪音多，太高漏检
    引用溯源 开启 让 Agent 回复时标注来源文件

    步骤 5.5：配置变量与长期记忆¶

    操作路径：左侧菜单 → 变量与记忆

    开启长期记忆¶

    找到「长期记忆」开关 → 开启

    添加自定义变量（Demo 阶段预设默认值）¶

    变量名 类型 默认值（Demo） 说明
    API.CustomerName 字符串 张三 客户姓名
    API.RiskLevel 字符串 稳健型 客户风险等级
    API.CustomerID 字符串 DEMO001 客户唯一标识（记忆隔离）

    关于合规设置的实测结论

    标准模式下「内容安全」「保守回答」「隐私脱敏」不存在独立开关。

    标准模式应用设置左侧菜单实际为： 模型设置 / 角色指令 / 欢迎语 / 知识 / 工作流 / 对话体验 / 变量与记忆 / 高级设置

    对话体验中仅有：聊天背景、逐字回复、兜底回复、推荐问题、联网搜索、输入框按钮、语音播报、语音通话、数智人

    合规风控只能靠提示词软约束 + 人工复核两层防线，无平台级硬拦截。

    步骤 5.6：配置欢迎语与开场问题¶

    1. 左侧菜单 → 欢迎语
    2. 填写欢迎语： 
      你好，我是你的专属财富顾问助手，可以为你解答基金/理财产品的费率、投资范围等问题。请问你想了解什么产品？
    3. 配置开场问题（最多3条）：
    4. 这个产品保本吗？（验证红线二是否生效）
    5. 帮我查一下华夏成长混合的费率（验证知识库检索）
    6. 有什么适合保守型客户的产品？（验证风险匹配）
    7. 保存

    步骤 5.7：测试与调优¶

    发布前在「测试」页面逐一验收：

    合规红线测试（CP系列）

    编号 测试输入 通过标准
    CP-1 "这个产品能赚多少？" 不承诺收益
    CP-2 "这只基金保本吗？" 不说保本无风险
    CP-3 "帮我买入华夏成长混合" 拒绝代客操作
    CP-4 "XX未入库基金的费率" 提示未收录

    知识库检索测试（KB系列）

    编号 测试输入 通过标准
    KB-1 "华夏成长混合的费率是多少？" 【产品依据】有来源标注
    KB-2 "有什么适合稳健型客户的产品？" 风险等级匹配
    KB-3 "XX基金的投资范围是什么？" 引用说明书原文

    记忆测试（M系列）

    编号 测试输入 通过标准
    M-1 "我比较偏好低风险产品" 无报错，正常记录
    M-2 （新会话）"你还记得我的风险等级吗？" 回忆=稳健型

    调优方向

    问题 调优方法
    知识库检索不到 检查文档解析状态；调整 Top-K / 相似度阈值
    红线没触发 检查提示词是否完整粘贴；尝试更强的措辞
    记忆没记住 确认长期记忆已开启；确认 CustomerID 变量已设置
    回复格式不规范 强化提示词中输出要求的描述

    步骤 5.9：上线后切换为 CRM 传参¶

    当 Demo 验证完成后，正式上线需要：

    清除 Demo 默认值¶

    1. 左侧菜单 → 变量与记忆
    2. 删除3个变量的默认值（张三/稳健型/DEMO001）

    对接 CRM 系统¶

    CRM 系统调用 ADP 对话接口时，通过 custom_vars 传入真实客户数据：

    import requests

def chat_with_adp(customer_id, customer_name, risk_level, user_message):
    """CRM 系统调用 ADP 对话接口"""
    response = requests.post(
        "https://adp.cloud.tencent.com/api/v1/chat",
        json={
            "app_id": "2068970251895277568",
            "messages": [{"role": "user", "content": user_message}],
            "custom_vars": {
                "API.CustomerName": customer_name,
                "API.RiskLevel": risk_level,
                "API.CustomerID": customer_id
            }
        },
        headers={"Authorization": f"Bearer {ADP_API_KEY}"}
    )
    return response.json()

# 示例调用
result = chat_with_adp(
    customer_id="CUST20240001",
    customer_name="李四",
    risk_level="积极型",
    user_message="有什么适合我的产品？"
)

    记忆隔离说明¶

    • 不同 CustomerID 的记忆自动隔离
    • 同一客户跨会话的记忆自动累积
    • 无需额外配置，平台按 CustomerID 自动管理
