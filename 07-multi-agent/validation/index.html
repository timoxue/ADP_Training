    TC-1 ~ TC-6 测试用例 | 验收检查清单 | 故障排查指南

    验收测试方案¶

    测试用例¶

    编号 输入 预期行为 验收标准
    TC-1 "分析宁德时代" 主 Agent → 4 子 Agent 并行 → 汇总报告 5 个模块都有输出，汇总完整，末尾有免责声明
    TC-2 "宁德时代最新新闻" 仅转交舆情专员 调度 Agent 正确判断，只转舆情子 Agent，不触发其他
    TC-3 "宁德时代的财务指标和行业对比" 同时转交财务分析师 + 行业分析师 并行触发，两个模块都有输出
    TC-4 "宁德时代有什么风险？" 转交风控专员 风险信号清单输出，高/中/低分级正确
    TC-5 "帮我分析一个不存在的公司代码 999999" 调度 Agent 判断不在能力范围内 提示用户输入正确的股票代码/名称
    TC-6 "对比宁德时代和比亚迪，并提取两家公司的关联交易" 转交财务分析师 + 风控专员 多维度并行，汇总报告含两个模块

    验收检查清单¶

    • [ ] TC-1：主 Agent 正确分发到 4 个子 Agent，汇总报告 5 个模块齐全
    • [ ] TC-1：汇总报告末尾有免责声明
    • [ ] TC-2：只触发舆情专员，其他子 Agent 不响应
    • [ ] TC-3：财务分析师和行业分析师同时被触发
    • [ ] TC-4：风控专员输出风险信号清单，分级正确
    • [ ] TC-5：主 Agent 提示用户输入正确的股票代码
    • [ ] TC-6：财务和风控两个模块同时触发，汇总完整
    • [ ] 所有测试：每个子 Agent 输出标注了数据来源
    • [ ] 所有测试：没有「不承诺收益」相关的违规输出

    常见问题排查¶

    问题 1：主 Agent 自己回答了，没有转交¶

    可能原因 排查方法 解决方法
    转交描述没配 检查子 Agent 配置页是否有转交描述 补填转交描述
    关键词不匹配 对比用户提问和转交描述关键词 扩充转交描述关键词
    主 Agent 提示词缺少分发指令 检查提示词是否有「转交」相关指令 强化提示词中的转交规则

    问题 2：只转交了一个子 Agent，其他没触发¶

    可能原因 排查方法 解决方法
    调度 Agent 判断为单维度需求 检查用户提问是否明确涉及多个维度 正常行为；如果是全面分析，用户应说「全面分析」
    提示词缺少多维度拆解指令 检查提示词是否有「一个需求可同时分发多个子 Agent」 在提示词中强化多维度并行分发逻辑

    问题 3：子 Agent 输出格式混乱¶

    可能原因 排查方法 解决方法
    提示词输出格式定义不够明确 检查提示词中的输出格式模板 强化格式模板，用表格和标题明确结构
    模型能力不足 检查模型版本 升级到更高版本模型

    问题 4：汇总报告缺模块¶

    可能原因 排查方法 解决方法
    某个子 Agent 没有返回结果 检查该子 Agent 的技能挂载 确认技能已正确挂载
    子 Agent 报错 查看调试日志 修复技能配置或提示词
    主 Agent 汇总时遗漏 检查主 Agent 提示词的汇总模板 强化汇总模板，明确 5 个模块必须齐全

    问题 5：任务流转到错误的子 Agent¶

    可能原因 排查方法 解决方法
    转交描述关键词重叠 对比各子 Agent 的转交描述 确保关键词互斥
    转交描述语义模糊 检查是否有歧义关键词 用更明确的专业术语替换
